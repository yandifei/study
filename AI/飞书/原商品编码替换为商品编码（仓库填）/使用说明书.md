# 剪贴板自动映射助手（静默替换版）  
## 使用说明书（最终用户版）  
**版本：v1.1（新增日志功能）**

---

### 1. 软件简介
本工具是一款后台静默运行的小程序，用于**自动将复制的“原商品编码”替换为 Excel 表中对应的“商品编码（仓库填）”**。  
您只需在任意地方（网页、文档等）选中一段文字，按下 `Ctrl+C`，粘贴出来的结果就会自动变成目标编码——整个过程无弹窗、无提示音，完全不影响您正常的复制粘贴操作。

**v1.1 新增功能**：程序会**自动记录每一次替换操作**（包括复制的内容、替换后的内容、操作时间及状态），并保存到本地日志文件中，方便您排查问题和追踪历史记录。

---

### 2. 系统要求
- **操作系统**：Windows 7 / 10 / 11（64位）
- **依赖环境**：无需安装 Python，直接运行 `.exe` 文件即可
- **必备文件**：  
  - 主程序 `ClipboardMapper.exe`（已打包好的可执行文件）  
  - Excel 数据源 `data.xlsx`（必须与 exe 放在**同一文件夹**下）

---

### 3. 安装与文件部署
将程序包解压后，您会得到以下文件结构（示例）：
```
您的文件夹/
├── ClipboardMapper.exe      ← 主程序
├── data.xlsx                ← 商品编码表（必须存在）
└── logs/                    ← 日志文件夹（程序启动时自动创建）
    └── clipboard_log.txt    ← 日志文件
```
> **请务必保证 `data.xlsx` 和 `ClipboardMapper.exe` 位于同一目录下**，否则程序无法找到数据文件。  
> 日志文件夹 `logs` 会在程序首次启动时自动生成，无需手动创建。

---

### 4. 准备 Excel 数据源
Excel 文件（`data.xlsx`）的格式必须严格按照以下要求制作：

| 原商品编码（索引列） | 商品编码（仓库填） | （其他列可随意添加） |
| -------------------- | ------------------ | -------------------- |
| smartband9pro        | Mibd9probz-p-99    | ...                  |
| phone13              | Ph13-black-64      | ...                  |

#### 必填列名（**必须完全一致，包括空格和括号**）：
- **第一列标题**：`原商品编码`  
- **第二列标题**：`商品编码（仓库填）`

> 您可以在这个表中任意增加其他列，程序只会识别这两列。  
> 数据可以包含空值，但程序会忽略空白行。

### 5. 启动与退出程序

#### 启动
- 双击 `ClipboardMapper.exe`，会弹出一个黑色命令行窗口（控制台），并显示类似信息：
  ```
  日志将保存至：C:\你的文件夹\logs\clipboard_log.txt
  映射表加载成功，共 123 条记录
  程序已启动，按 ESC 键退出。
  按下 Ctrl+C 将自动映射（仅匹配格式的文本）
  ```
- **请勿关闭这个窗口**（它表示程序正在后台运行）。您可以将其最小化，不影响使用。

#### 退出
- 在命令行窗口上按键盘 **`ESC`** 键，程序即安全退出，窗口自动关闭。

### 6. 日常使用流程
1. **在任意界面（网页、Excel、文档等）中**，选中一段文字（例如 `smartband9pro`）。  
2. 按下 **`Ctrl+C`** 复制。  
   - 程序会在后台 **静默** 完成以下操作（毫秒级）：  
     a. 检查您复制的内容是否符合“原商品编码”的格式（默认只允许 **字母、数字、下划线、短横线**，例如 `smartband9pro` 符合要求）。  
     b. 若格式匹配，则在 Excel 表中查找该编码。  
     c. 如果找到 → 将剪贴板内容替换为对应的“商品编码（仓库填）”，并记录一条“成功”日志；  
        如果未找到 → 剪贴板内容变为提示文字 `没有找到该数据，请更新“Excel文件”或手动查找`，并记录一条“失败”日志。  
     d. 若格式不匹配（例如复制了 `2025-06-24` 或 `产品名称`），则程序**不做任何处理**，保留原始复制内容，**且不记录日志**（仅针对符合格式的内容记录）。
3. 在需要粘贴的位置按下 **`Ctrl+V`**，粘贴出来的就是替换后的结果。

#### 举个例子
| 您的操作                             | 粘贴出来的结果                                | 日志记录情况 |
| ------------------------------------ | --------------------------------------------- | ------------ |
| 复制 `smartband9pro`（存在于表中）   | `Mibd9probz-p-99`（自动替换）                 | 记录成功日志 |
| 复制 `unknown_code`（不存在于表）    | `没有找到该数据，请更新“Excel文件”或手动查找` | 记录失败日志 |
| 复制 `产品型号：ABC-123`（格式不符） | `产品型号：ABC-123`（原样不变）               | 不记录日志   |

### 7. 注意事项（重要！）

#### 7.1 格式校验规则（匹配模式）
- 程序默认只对 **纯字母、数字、下划线 `_` 和短横线 `-`** 组成的编码进行替换（正则表达式 `^[A-Za-z0-9\-_]+$`）。  
- 如果您的商品编码包含其他字符（如空格、点 `.`、括号等），程序将**忽略**本次复制，不会进行替换。  
- **若要修改匹配规则**，请联系开发者或自行修改代码中的 `PATTERN` 变量后重新打包。

#### 7.2 Excel 文件更新
- 若您增减了商品数据，**必须重启程序**才能加载最新的映射表。  
  （程序仅在启动时读取一次 Excel，不自动热更新）

#### 7.3 程序运行时不要移动或删除 `data.xlsx`
- 程序运行时若数据文件被移走或重命名，会导致替换失败。请确保文件始终在 exe 同级目录下。

#### 7.4 关于“商品编码（仓库填）”为空的情况
- 如果某行“商品编码（仓库填）”为空，当匹配到该行时，剪贴板会被替换为空字符串（相当于清空剪贴板）。请务必检查数据完整性。

#### 7.5 程序仅监听 `Ctrl+C`，不干扰其他快捷键
- 本程序不会影响您原有的复制操作，也不会占用系统资源（后台占用内存约 30MB）。

#### 7.6 日志文件会持续增长（v1.1 新增）
- 每次成功替换或失败替换都会写入一条日志，长期使用后日志文件可能变大。建议**定期清理** `logs` 文件夹中的旧日志，或手动删除 `clipboard_log.txt`（程序下次启动会自动重建）。  
- 日志文件默认保存在程序所在目录的 `logs` 子文件夹中，您也可以直接打开查看。

### 8. 常见问题排查

| 问题现象                    | 可能原因及解决办法                                                                                          |
| --------------------------- | ----------------------------------------------------------------------------------------------------------- |
| 双击 exe 后闪退             | ① `data.xlsx` 不存在或列名不正确 → 检查文件位置和列名；<br>② Excel 文件被其他程序占用 → 关闭 Excel 后重试。 |
| 复制了正确的编码，但没反应  | ① 编码包含空格或特殊字符（如 ` `、`.`）→ 格式不匹配，无法替换；<br>② 未重启程序加载新数据 → 重新启动程序。  |
| 粘贴出来是“没有找到该数据…” | 编码确实不在表中 → 请添加对应行或检查是否存在大小写、空格差异（程序区分大小写）。                           |
| 复制普通文字也被替换了      | 检查普通文字是否巧合符合正则格式（如全英文数字）→ 若不需要替换，请修改代码中的正则规则。                    |
| 想查看历史替换记录          | 打开 `logs/clipboard_log.txt` 即可查看每次操作的时间、原始内容、替换内容和状态。                            |

### 9. 高级定制（仅供开发者参考）
本程序配置部分位于源代码顶部，若需要修改，请编辑以下变量后重新打包：
```python
EXCEL_PATH  = rf"{get_root()}/data.xlsx"    # 可改为绝对路径或自定义文件名
SHEET_NAME  = 0                             # 工作表索引（0表示第一个表）
COL_ORIGINAL = "原商品编码"                  # 索引列名（务必与表头一致）
COL_TARGET   = "商品编码（仓库填）"         # 目标列名
PATTERN      = r'^[A-Za-z0-9\-_]+$'         # 自定义正则匹配规则
FAIL_TEXT    = '没有找到该数据，请更新“Excel文件”或手动查找'
# 日志相关配置（v1.1新增）
LOG_DIR      = get_root() / "logs"          # 日志文件夹路径
LOG_FILE     = LOG_DIR / "clipboard_log.txt" # 日志文件名
```
> 普通用户**无需修改**，直接使用默认配置即可满足大多数场景。

### 11. 日志文件说明（v1.1 新增）
为了帮助您追踪替换行为和排查问题，程序会在 `logs/clipboard_log.txt` 中记录每一次有效的替换操作（即复制内容符合格式校验的操作）。

#### 日志内容格式
每条记录占一行，字段之间用 **制表符**（Tab）分隔，包含以下四列：

| 时间戳（精确到毫秒）    | 原始复制内容  | 替换后内容                                  | 状态    |
| ----------------------- | ------------- | ------------------------------------------- | ------- |
| 2026-06-24 15:30:12.345 | smartband9pro | Mibd9probz-p-99                             | SUCCESS |
| 2026-06-24 15:30:15.678 | unknown_code  | 没有找到该数据，请更新“Excel文件”或手动查找 | FAILED  |

#### 如何查看日志
- 直接使用记事本或任何文本编辑器打开 `logs/clipboard_log.txt`。
- 若需导入 Excel 进一步分析，可使用“数据”→“自文本/CSV”导入，并选择制表符作为分隔符。

#### 日志清理
- 该日志文件会持续追加写入，建议定期（如每月）删除或归档旧日志，避免占用过多磁盘空间。删除日志文件不影响程序正常运行，下次替换时会自动重建。

### 一、 需求描述（给开发者的规格书）

> **项目名称**：剪贴板自动映射助手（静默替换版）
> **核心目标**：通过监听 `Ctrl+C` 操作，自动从本地 Excel 中检索并替换剪贴板内容为指定的商品编码，无任何弹窗提示，完全静默运行。
>
> **【输入与数据源】**
> 1. **监听触发**：全局监听 Windows 系统的键盘按键 `Ctrl+C`。
> 2. **数据源**：预加载指定的本地 `xlsx` 或 `xls` 表格文件。
> 3. **表格列映射**：
>    - 索引列：`原商品编码`
>    - 目标替换列：`商品编码（仓库填）`
>
> **【执行逻辑（依次执行）】**
> 1. **格式预校验**（核心防误触）：当检测到 `Ctrl+C` 时，读取剪贴板中的文本。**只有当该文本符合“原商品编码”的特定格式时**（例如：固定字符长度、或特定的前缀后缀，需在代码中配置正则匹配），才进入下一步查找。否则，**放弃替换**，保持剪贴板原内容不变，以确保不影响正常复制。
> 2. **索引查找**：通过预校验后，在表格的 `原商品编码` 列中尝试精确匹配剪贴板文本。
> 3. **分支处理**：
>    - **✅ 匹配成功**：读取目标列 `商品编码（仓库填）` 的内容，**直接覆盖**系统剪贴板。
>    - **❌ 匹配失败**：将系统剪贴板的内容**强制替换**为指定文字：`没有找到该数据，请更新“Excel文件”或手动查找`。
> 4. **静默处理**：整个流程在后台毫秒级完成，**无需弹窗**、**无需提示音**，用户只需在操作完成后直接按下 `Ctrl+V` 粘贴即可看到结果。

### 二、 用户实际工作流程（操作与后台对照）

为了方便你理解整个过程，我把你的操作和程序在后台做的事对应起来：

| 你的操作（前台） | 程序后台行为（3毫秒内完成） |
| :--- | :--- |
| **1. 启动程序** | 程序静默运行，读取表格并建立内存索引，在系统任务栏托盘处挂起监听。 |
| **2. 在网页中，选中一个“原商品编码”**（如：`smartband9pro...`） | 等待监听。 |
| **3. 按下 `Ctrl + C`** | **①** 系统先完成复制；**②** 程序立即截获剪贴板内容；**③** 判断格式是否符合；**④** 去表格中搜索。 |
| **4. 发现表格中有该行数据** | 程序自动提取出目标列 `商品编码（仓库填）`（如：`Mibd9probz-p-99`），并将这个新内容**覆盖**到剪贴板中。 |
| **5. 回到网页输入框中，按下 `Ctrl + V`** | 粘贴出来的直接就是 `Mibd9probz-p-99`，无需你再翻找表格。 |
| **6. （异常情况）你复制了一个表格里不存在的“原商品编码”** | 程序查表未找到，立刻将剪贴板内容强制替换为 `没有找到该数据，请更新“Excel文件”或手动查找`。 |
| **7. 按 `Ctrl + V` 粘贴** | 输入框出现提示文字，告知你当前复制的内容未录入表格。 |
| **8. 你在网页复制了标题、日期等其他普通文本** | 由于程序发现文本**不符合“原商品编码”的特征格式**，程序**什么都不做**，剪贴板内容保持不变，保证你的日常复制不受影响。 |

### 💡 开发者实施补充建议
1. **语言与技术推荐**：依然是推荐使用 **Python**（结合 `keyboard` 库监听 Ctrl+C，`pyperclip` 处理剪贴板，`pandas` 读取并缓存 Excel）。
2. **进程驻留**：程序需打包为 `.exe` 后台运行。可以增加一个选项，让程序在启动时最小化到右下角托盘。
3. **替换时机**：程序拦截到 `Ctrl+C` 时，**必须延迟几十毫秒**再执行替换（使用 `time.sleep(0.05)`），因为系统需要时间把内容写入剪贴板，如果程序立刻读取，可能读到的是上一次剪贴板的内容。

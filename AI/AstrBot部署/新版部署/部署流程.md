## 提取核心部署文件
因为我不需要源代码构建，所以我其实仅仅需要的就是compose文件而已，所以直接提取核心文件
`shipyard-neo\deploy\docker`下的三个`yaml`文件就是核心文件
可以对它们进行一定的整合`yaml`文件，联合部署进行统一管理
理论上可以把这三个`yaml`文件整合为1个文件，但是针对于`config.yaml`文件是个例外，他是`bay`服务的依赖文件，正常时在`bay`中的，但是因为配置项太多，所以单独提取出来，方便修改和更新。

整合`docker-compose.with-astrbot.yaml`和`docker-compose.yaml`文件后加上`napcat`服务就是
`docker-compose.astrbot_napcat_shipyard-neo.yaml`文件了。
最后统计就是`docker-compose.astrbot_napcat_shipyard-neo.yaml`及其`bay`服务的依赖文件`config.yaml`了


### 1. **设置强随机 API Key**
```yaml
security:
  api_key: "CHANGE-ME"   # 改为实际密钥
```
生成方法（在宿主机执行）：
```bash
openssl rand -hex 32
```
将输出的一长串字符替换进去。
**这种方式是自定义固定密钥，下面呢的Access Token留空无效，因为只有当你什么都不干的时候才生成密钥在credentials.json，下面的Access Token留空才有效，所以这里改了实际密钥就是固定的了，你得手动输入Access Token**


1. 访问 `http://你的IP:6185`，登录（默认 `astrbot` / `astrbot`）
2. 进入 **AI 配置** → **Agent Computer Use**
3. 设置：
   - Computer Use Runtime = `sandbox`
   - 沙箱环境驱动器 = `shipyard_neo`
   - Endpoint = `http://bay:8114`（如果一体化部署在同一 compose 网络中）
   - Access Token = **留空**（AstrBot 会通过挂载的 `bay-data` 卷自动发现）
4. 保存并测试。

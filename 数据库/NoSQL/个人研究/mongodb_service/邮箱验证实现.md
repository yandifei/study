# 邮箱验证码实现

> 模块: `services/auth/verification.py`  
> 依赖: `services/database/redis.py`（Redis 客户端）、`utils/message_util.py`（邮件发送）

---

## 一、架构

```
POST /auth/send-code { email }
        │
        ▼
router.py  send_verification_code()
        │
        ▼
verification.py  send_code(email)
        │
        ├── ① check_send_cooldown(email)   → Redis: EXISTS email_cooldown:{email}
        ├── ② generate_code()              → "583921"
        ├── ③ save_verification_code()     → Redis: SET email_code:{email} TTL 300s
        ├── ④ send_verification_email()    → utils/message_util.py (QQ SMTP)
        └── ⑤ set_send_cooldown(email)     → Redis: SET email_cooldown:{email} TTL 60s
```

调用关系：

```
router.py  ──→  verification.py  ──→  utils/message_util.py  ──→  QQ邮箱 SMTP
(HTTP层)        (业务逻辑层)          (邮件发送层)                 (外部服务)
                     │
                     └──→  services/database/redis.py  ──→  Redis
                           (存储层)                          (缓存)
```

## 二、Redis Key 设计

| Key | 值 | TTL | 用途 |
|-----|-----|-----|------|
| `email_code:{email}` | 6位验证码，如 `"583921"` | 300s（5分钟） | 存储验证码，登录时校验 |
| `email_cooldown:{email}` | `"1"` | 60s | 发送冷却标记，防刷 |

示例：

```
email_code:3058439878@qq.com    →  "583921"    EX 300
email_cooldown:3058439878@qq.com →  "1"         EX 60
```

**为什么分开两个 Key？** 验证码和冷却期生命周期不同：验证码 5 分钟有效，冷却期只要 60 秒。分开存储各自独立过期，互不影响。

## 三、函数清单

| 函数 | 类型 | 可见性 | 说明 |
|------|------|--------|------|
| `_code_key(email)` | 同步 | 私有 | 拼接验证码 Redis key |
| `_cooldown_key(email)` | 同步 | 私有 | 拼接冷却期 Redis key |
| `generate_code()` | 同步 | **公开** | 生成6位随机数字，`random.choices('0123456789', k=6)` |
| `save_verification_code(email, code)` | 异步 | 公开 | 验证码写入 Redis，TTL 由环境变量控制 |
| `verify_and_consume_code(email, code)` | 异步 | **公开** | 校验验证码，匹配则删除（一次性） |
| `check_send_cooldown(email)` | 异步 | 公开 | 检查是否在60s冷却期内 |
| `set_send_cooldown(email)` | 异步 | 公开 | 设置60s冷却标记 |
| `send_code(email)` | 异步 | **公开** | 编排完整流程，返回 `(bool, str)` |

## 四、核心流程详解

### 4.1 发送验证码 — `send_code(email)`

```
send_code("3058439878@qq.com")
    │
    ├── ① check_send_cooldown
    │        EXISTS email_cooldown:3058439878@qq.com
    │        ├── 存在 → 返回 (False, "请 45 秒后再试")
    │        └── 不存在 → 继续
    │
    ├── ② generate_code
    │        返回 "583921"
    │
    ├── ③ save_verification_code
    │        SET email_code:3058439878@qq.com "583921" EX 300
    │
    ├── ④ send_verification_email("3058439878@qq.com", "583921")
    │        调用 utils/message_util.py → QQ邮箱 SMTP → 发送 HTML 邮件
    │        模板: templates/email_template.html
    │        ├── 成功 → 继续
    │        └── 异常 → 返回 (False, "邮件发送失败: ...")
    │
    └── ⑤ set_send_cooldown
             SET email_cooldown:3058439878@qq.com "1" EX 60
             返回 (True, "验证码已发送")
```

**冷却期判断在生成验证码之前**，避免恶意用户无成本消耗 Redis 和邮件配额。

### 4.2 校验验证码 — `verify_and_consume_code(email, code)`

```
verify_and_consume_code("3058439878@qq.com", "583921")
    │
    ├── GET email_code:3058439878@qq.com
    │   ├── None → 返回 False（验证码不存在或已过期）
    │   └── "583921" → 继续
    │
    ├── "583921" != "583921"?
    │   ├── 不匹配 → 返回 False
    │   └── 匹配 → 继续
    │
    └── DEL email_code:3058439878@qq.com  ← 一次性消费
        返回 True
```

**为什么校验后立即删除？** 防止同一验证码被多次使用。即使攻击者截获了验证码，也只能用一次。

## 五、Router 层的调用

`services/auth/router.py` 中 `/auth/send-code` 端点：

```python
@router.post("/send-code")
async def send_verification_code(request: SendCodeRequest):
    success, message = await send_code(request.email)
    if not success:
        raise HTTPException(
            status_code=429 if "秒后再试" in message else 500,
            detail=message,
        )
    return {"code": 200, "msg": message, "data": None}
```

**请求/响应：**

```json
// 请求
POST /auth/send-code
{ "email": "3058439878@qq.com" }

// 成功
{ "code": 200, "msg": "验证码已发送", "data": null }

// 冷却期内
HTTP 429
{ "detail": "请 45 秒后再试" }

// 邮件发送失败
HTTP 500
{ "detail": "邮件发送失败: ..." }
```

## 六、环境变量

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `VERIFICATION_CODE_TTL` | `300` | 验证码有效期（秒），默认 5 分钟 |
| `COOLDOWN_TTL` | `60` | 发送冷却期（秒），默认 60 秒 |

这两个变量当前未在 `config/.env` 中设置，使用默认值。如需自定义，在 `.env` 中添加即可。

## 八、单元测试

`verification.py` 内置 `__main__` 可直接测试（需要 Redis 运行）：

```bash
python services/auth/verification.py
# 向 3058439878@qq.com 发送验证码
```

或手动测试：

```python
import asyncio
from services.auth.verification import send_code, verify_and_consume_code

# 发送
success, msg = asyncio.run(send_code("test@qq.com"))
print(msg)

# 校验（需要从 Redis 或邮箱拿到真实验证码）
ok = asyncio.run(verify_and_consume_code("test@qq.com", "123456"))
print("校验结果:", ok)
```

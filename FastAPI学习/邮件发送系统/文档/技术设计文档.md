## 2. 技术选型

### 2.1 技术栈总览

| 类别 | 技术 | 版本 | 选型理由 |
|------|------|------|----------|
| **语言** | Python | 3.14 | 类型标注完善，异步支持好 |
| **Web 框架** | FastAPI | 0.136+ | 高性能异步、自动 OpenAPI 文档、Pydantic 集成 |
| **ASGI 服务器** | Uvicorn | 0.49+ | 轻量、高性能 |
| **数据校验** | Pydantic | 2.13+ | EmailStr、类型安全 |
| **数据库** | SQLite3 | 内置 | 零配置、无需额外服务、适合单机部署 |
| **邮件协议** | SMTP (SSL) | — | QQ 邮箱 SMTP 服务 |
| **前端** | HTML5 + CSS3 + Vanilla JS | — | 无框架依赖、零构建步骤 |
| **前端字体** | Orbitron + DM Sans + JetBrains Mono | — | Google Fonts CDN |
| **日志** | logging + colorlog | 6.10+ | 彩色控制台输出 |
| **配置管理** | TOML / YAML / .env | — | 多层级配置覆盖 |
| **包管理** | uv | — | 快速依赖解析 |

### 2.2 关键设计决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 数据库 | SQLite（非 MongoDB/MySQL） | 单机部署，零运维成本 |
| 前端框架 | 无框架 Vanilla JS | 项目规模适中，避免构建工具链复杂度 |
| 模板引擎 | 字符串替换（非 Jinja2） | 邮件模板简单，减少依赖 |
| 调度方式 | 后台线程 + 轮询（非 Celery） | 避免消息队列依赖，简化部署 |
| 邮件图片 | 内嵌 CID (Content-ID) | 避免外部图片链接被邮件客户端屏蔽 |

### 3.2 核心类/函数说明

#### message_util.py — 邮件发送层

```
_build_and_send(email, subject, html) → bool
    ├── 构建 MIMEMultipart('related')
    ├── 附加 HTML 正文
    ├── 附加 Logo 图片 (cid:logo)
    ├── SMTP_SSL 连接 smtp.qq.com:465
    └── 返回 True/False

send_verification_email(email, verification_num) → bool
send_password_reset_email(email, user_name, reset_link, ...) → bool
... (共 11 个 send_* 函数)
```

**设计模式**：模板方法模式 — 所有发送函数共享 `_build_and_send` 公共逻辑，差异仅在于模板选择和变量替换。

#### email_task_db.py — 数据访问层

```
init_db()                           # 建表
insert_task(email, time, code, vars) # 单条插入
insert_tasks_batch(tasks[])         # 批量插入
get_due_tasks()                     # 查询到期任务
get_all_tasks(status)               # 查询所有（可按状态过滤）
mark_task_sent(id)                  # 标记已发送
mark_task_failed(id)                # 标记失败
delete_task(id)                     # 删除任务
get_task_count(status)              # 统计计数

TEMPLATE_DEFINITIONS                # 11 套模板字段定义（供 API 返回）
```

**线程安全**：所有写操作使用 `threading.Lock` 保护。

#### email_dispatcher.py — 调度分发层

```
DISPATCH_MAP = {
    "VERIFY_CODE":    {"fn": send_verification_email,      "arg_map": [...]},
    "PASSWORD_RESET": {"fn": send_password_reset_email,     "arg_map": [...]},
    ... (11 条映射)
}

dispatch_email(email, template_code, template_vars) → (bool, str)
    └── 查 DISPATCH_MAP → 构建参数列表 → 调用对应函数
    └── 返回 (成功标志, 错误消息)

send_due_emails() → int
    └── get_due_tasks() → 逐条 dispatch_email → 更新状态 + 记录错误

send_single_task(task_id) → bool

start_scheduler(interval_seconds=60) → 启动后台线程
stop_scheduler()                     → 停止后台线程
```

### 3.3 email_task_db.py — 数据访问层 (v1.1 更新)

**新增功能**：
- `error_msg` 列：存储 SMTP 错误详情（最长 500 字符）
- `mark_task_failed(task_id, error_msg)`：记录失败原因
- `reset_failed_tasks()`：将所有失败任务重置为 pending
- `reset_task_to_pending(task_id)`：重置单个任务
- `delete_tasks_batch(task_ids[])`：批量删除
- `delete_tasks_by_status(status)`：按状态清空

## 4. 数据库设计

> 完整数据库设计详见 **[《数据库设计文档》](./数据库设计文档.md)**，包含选型分析、E-R 图、范式分析、字段详解、WAL 模式、线程安全设计等。

### 4.1 概要

```
┌──────────────┐
│  email_tasks │
├──────────────┤
│ PK id        │── 自增主键
│    user_email│── 收件人邮箱
│    send_time │── 计划发送时间 (ISO 8601)
│    template  │── 模板代码 (FK → TEMPLATE_DEFINITIONS)
│    vars      │── 模板变量 (JSON)
│    status    │── pending | sent | failed
│    created_at│── 创建时间
│    sent_at   │── 实际发送时间 (可空)
└──────────────┘
```

### 4.2 DDL

```sql
CREATE TABLE IF NOT EXISTS email_tasks (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    user_email      TEXT    NOT NULL,
    send_time       TEXT    NOT NULL,
    template_code   TEXT    NOT NULL,
    template_vars   TEXT    NOT NULL DEFAULT '{}',
    status          TEXT    NOT NULL DEFAULT 'pending',
    created_at      TEXT    NOT NULL,
    sent_at         TEXT
);
```

### 4.3 索引策略

| 索引 | 用途 |
|------|------|
| `status + send_time` | 调度器查询到期任务（复合条件） |
| `status` | 统计查询过滤 |

> 注：SQLite 在 WAL 模式下自动优化读操作，当前数据量无需显式建索引。

### 4.4 状态机

```
  [创建] → pending ──[发送成功]──→ sent
                │
                └──[发送失败]──→ failed
```

## 7. 调度器设计

### 7.1 调度流程

```
服务启动 (lifespan startup)
  │
  ├── init_db()                  # 初始化数据库表
  ├── send_due_emails()          # 立即处理遗留到期任务
  └── start_scheduler(60s)       # 启动后台线程
        │
        └── while not stopped:
              ├── get_due_tasks()      # SELECT pending WHERE send_time <= now
              ├── for each task:
              │     ├── dispatch_email()
              │     ├── mark_sent() / mark_failed()
              └── sleep(60s)
```

### 7.2 调度器参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `interval_seconds` | 60 | 扫描间隔 |
| 线程类型 | daemon | 主进程退出时自动终止 |
| 线程安全 | `threading.Event` | 可安全停止 |

## 8. 安全设计

| 层面 | 措施 |
|------|------|
| SMTP 凭据 | 仅通过环境变量 `SENDER_EMAIL` / `SMTP` 注入 |
| SQL 注入 | 全部使用参数化查询（`?` 占位符） |
| XSS | 前端 `escapeHtml()` 转义用户输入 |
| 邮箱校验 | Pydantic EmailStr 基于 RFC 5322 |
| 线程安全 | SQLite WAL 模式 + `threading.Lock` |

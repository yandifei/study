## 2. 邮件模板系统

### 2.1 模板总览

系统内置 **11 种** 邮件渲染模板，覆盖事务类、营销类、生命周期类三大邮件场景。

#### 事务类模板（8 种）

| 模板代码 | 名称 | 用途 | 特色元素 |
|----------|------|------|----------|
| `VERIFY_CODE` | 验证码邮件 | 注册/登录验证 | 大号验证码展示卡片 |
| `PASSWORD_RESET` | 密码重置 | 密码重置确认 | 重置按钮 + 安全信息（IP/浏览器/有效期） |
| `PASSWORD_CHANGED` | 密码修改通知 | 修改成功通知 | 纯通知型，无操作按钮 |
| `LOGIN_ALERT` | 登录提醒 | 异地/新设备登录 | 红色警示头部 + 登录详情卡片 |
| `EMAIL_CHANGE` | 邮箱变更确认 | 旧邮箱确认 | 橙色主题 + 取消变更按钮 |
| `ORDER_CONFIRM` | 订单确认 | 下单/支付成功 | 绿色主题 + 商品明细表 + 合计金额 |
| `SHIP_NOTIFY` | 物流发货 | 发货通知 | 蓝色主题 + 快递单号 + 查询按钮 |
| `REFUND_STATUS` | 退款进度 | 退款状态变更 | 紫色主题 + 退款状态卡片 |

#### 营销/内容类模板（2 种）

| 模板代码 | 名称 | 用途 | 特色元素 |
|----------|------|------|----------|
| `PROMO_BATCH` | 促销优惠券 | 营销推广 | 红色主题 + 优惠码卡片 + Banner + 商品推荐 + 退订链接 |
| `NEWSLETTER` | 周刊资讯 | 内容推送 | 绿色主题 + 文章列表卡片 + 退订链接 |

#### 生命周期类模板（1 种）

| 模板代码 | 名称 | 用途 | 特色元素 |
|----------|------|------|----------|
| `ACCOUNT_WELCOME` | 注册欢迎 | 新用户欢迎 | 入门步骤列表 + 可选的邮箱验证按钮 |

### 2.2 模板变量系统

每个模板有独立的变量定义。变量分为两类：

**简单变量**：直接在输入框中填写
- 示例：`user_name` = `"张三"`、`verification_code` = `"123456"`

**复合变量（JSON）**：以 JSON 数组/对象格式填写
- 示例：`items` = `[{"name":"耳机","price":"199","quantity":"1"}]`
- 示例：`articles` = `[{"title":"标题","summary":"摘要","link":"https://..."}]`

> 完整变量参考见 **[渲染模板类型.md](./渲染模板类型.md)**

### 2.3 动态表单

选择模板后，系统自动渲染对应的变量输入表单：

1. **描述提示框**：蓝色左边框，展示模板用途描述
2. **必填标识**：红色 `*` 标记必填字段
3. **输入类型匹配**：文本/数字/URL/邮箱/JSON 自动匹配对应 input type
4. **JSON 字段**：自动使用 textarea + monospace 字体，placeholder 给出格式示例

## 3. 任务创建与管理

### 3.1 创建任务流程

```
选择模板 → 填写变量 → 输入收件人邮箱 → 校验邮箱 → 设置定时(可选) → 创建任务
```

### 3.2 输入收件人邮箱

**手动输入**：
- 在 textarea 中每行输入一个邮箱地址
- 实时显示已输入邮箱计数
- 空行自动忽略

**TXT 文件导入**：
- 点击「📂 TXT导入」按钮
- 选择 `.txt` 文件（一行一个邮箱，支持 `\r\n` 和 `\n` 换行）
- 若文本框已有内容，追加到末尾
- 导入后自动更新计数

### 3.3 定时发送

| 选项 | 行为 |
|------|------|
| **关闭（默认）** | 任务创建后立即进入调度队列，send_time = 当前时间 |
| **开启** | 显示日期时间选择器，默认值为当前时间 + 1 小时 |

- 定时任务的 send_time 为未来时间，后台调度器到期后自动发送
- 支持任意未来时间，不受限制

### 3.4 提交确认

- 点击「🚀 创建任务」按钮
- 快捷键：`Ctrl + Enter`
- 提交前自动调用后端 Pydantic 校验
- 校验通过：任务入库，重置表单，刷新列表
- 校验不通过：展开校验面板显示逐行错误

## 4. 批量操作与快捷功能

### 4.1 快速操作栏

位于任务列表顶部，提供 5 种批量操作：

| 按钮 | 功能 | 确认 |
|------|------|------|
| ⚡ **一键发送全部** | 立即发送所有 status=pending 的任务（忽略定时） | 需要 |
| 🔄 **重试全部失败** | 将所有 failed 任务重置为 pending → 立即发送 | 需要 |
| 🗑 **批量删除** | 删除所有勾选的任务 | 需要 |
| 🧹 **清空已完成** | 删除所有 status=sent 的记录 | 需要 |

### 4.2 复选框选择

- 每行左侧有复选框
- 表头复选框 = 全选/取消全选
- 选中后「已选 N 项」计数显示
- 「批量删除」按钮仅在选中 ≥1 项时可用

### 4.3 单任务操作

每行操作列根据状态显示不同按钮：

| 状态 | 可用操作 |
|------|----------|
| `pending`（待发送） | ▶ 立即发送 / ✕ 删除 |
| `sent`（已发送） | ✕ 删除 |
| `failed`（失败） | 🔄 重试 / ✕ 删除 |

## 5. 邮箱校验系统

### 5.1 双阶段校验

**阶段一：手动校验（前端触发）**

1. 输入邮箱后点击「🔍 校验」按钮
2. 调用 `POST /api/email-tasks/validate-emails`
3. 后端使用 Pydantic `EmailStr._validate()` 逐条校验（RFC 5322 标准）

**阶段二：提交时强校验**

1. 点击「创建任务」时自动校验
2. 不通过则阻止提交，展示错误详情
3. 通过后方可入库

### 5.2 校验结果展示

校验结果以面板形式显示在邮箱文本框下方：

```
┌────────────────────────────────────────┐
│ ✅ 全部通过 — 共 3 个邮箱均合法         │  ← 绿色背景（全部通过）
│ 或                                     │
│ ❌ 2 个邮箱格式不合法 / 1 个通过        │  ← 红色背景（有错误）
├────────────────────────────────────────┤
│ #1  ✓  user@qq.com                    │  ← 绿色行
│ #2  ✗  bad-email     邮箱格式不合法    │  ← 红色行 + 错误原因
│ #3  ✗  (空)          邮箱为空          │  ← 红色行 + 具体提示
└────────────────────────────────────────┘
```

**校验规则**：
- 邮箱为空 → "邮箱为空"
- 不符合 RFC 5322 → "邮箱格式不合法"
- 校验异常 → "邮箱校验异常"
- Pydantic EmailStr 会拒绝：缺少 `@`、缺少域名、非法字符、连续 `@@` 等

## 6. 定时调度系统

### 6.1 调度机制

```
服务启动
  │
  ├── init_db()              # 建表 + 向前兼容迁移
  ├── send_due_emails()      # 处理遗留到期任务
  └── start_scheduler(60s)   # 启动后台 daemon 线程
        │
        └── 循环 (每 60 秒):
              ├── SELECT * WHERE status='pending' AND send_time <= NOW
              ├── 逐条 dispatch_email()
              │     ├── 成功 → status='sent', sent_at=NOW
              │     └── 失败 → status='failed', error_msg='...', sent_at=NOW
              └── sleep(60)
```

### 6.2 调度器特性

| 特性 | 说明 |
|------|------|
| 线程类型 | daemon（主进程退出自动终止） |
| 扫描间隔 | 60 秒（可配置） |
| 启动扫描 | 服务启动立即执行一次 |
| 失败重试 | 手动触发（按钮或 API） |
| 状态持久化 | 每次发送结果立即写入 SQLite |

### 6.3 发送失败处理

- SMTP 异常（认证失败/网络错误/超时等）被捕获
- 错误信息截取前 500 字符存入 `error_msg` 字段
- 任务状态标记为 `failed`
- 用户可通过 UI 查看错误详情（点击失败徽章）
- 支持单个重试或批量重试

## 7. 任务详情与错误追踪

### 7.1 行展开详情

**操作**：点击任务行（非复选框/按钮区域）

**展开内容**：
- 📧 收件人邮箱
- 🕐 创建时间
- 📦 模板变量（JSON 格式化展示，monospace 字体）
- 📤 实际发送时间（仅已处理的任务）
- ❌ 错误详情（仅失败任务，红色边框高亮）

**再次点击**：收起详情

### 7.2 错误详情弹窗

**操作**：点击失败任务的红色状态徽章

**弹窗内容**：
- 任务 ID
- 收件人邮箱
- 使用的模板
- 创建时间
- 完整错误消息（红色高亮卡片）
- 🔄 重试发送按钮

**关闭**：点击遮罩层 / 关闭按钮 / ESC

### 7.3 状态筛选

| 操作 | 效果 |
|------|------|
| 点击「全部」标签 | 显示所有任务 |
| 点击「待发送」标签 | 仅显示 pending |
| 点击「已发送」标签 | 仅显示 sent |
| 点击「失败」标签 | 仅显示 failed |
| 点击统计卡片 | 等同于点击对应标签 |

## 8. 主题切换

### 8.1 双主题

| 主题 | 背景色 | 卡片色 | 文字色 | 强调色 |
|------|--------|--------|--------|--------|
| 🌙 **夜间模式** | `#0d0a1a` 深紫黑 | `#231d42` 暗紫 | `#e8e4f0` 浅白 | `#7b6cd1` 薰衣草紫 |
| ☀️ **白天模式** | `#f0edf7` 浅紫白 | `#ffffff` 纯白 | `#1e1838` 深紫 | `#6c5ce7` 深紫 |

### 8.2 切换方式

- 点击 Header 中的 🌙 / ☀️ 按钮
- 选择自动持久化到 `localStorage`
- 刷新页面 / 重启浏览器后保持

### 8.3 技术实现

使用 CSS 属性选择器 `html[data-theme="dark"]` / `html[data-theme="light"]`，所有颜色通过 CSS 变量（如 `var(--bg-card)`）引用。一套 HTML + CSS 即可覆盖两种主题，无需加载额外的样式表。

## 9. 快捷键与效率操作

| 快捷键 | 功能 |
|--------|------|
| `Ctrl + Enter` | 在表单任意输入框内按下 → 提交创建任务 |
| `ESC` | 关闭 Modal 弹窗 |

### 效率提示

- **批量导入**：准备 `.txt` 文件（一行一邮箱），点击「📂 TXT导入」批量添加
- **快速筛选**：点击顶部统计卡片直接过滤对应状态
- **二次确认**：所有删除操作都有 Modal 确认对话框，防止误操作
- **一键发送**：无需逐个点击，使用「一键发送全部」批量处理所有待发任务
- **失败处理**：点击失败徽章查看原因 → 点击「🔄 重试」修复后重新发送

## 附录：完整 API 响应格式

所有 API 遵循统一格式：

```json
{
  "code": 200,
  "msg": "操作描述信息",
  "data": { /* 具体数据 */ }
}
```

| code | 含义 |
|------|------|
| 200 | 成功 |
| 400 | 客户端错误（参数不合法） |
| 404 | 资源不存在 |
| 500 | 服务端错误（SMTP 失败等） |

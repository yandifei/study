# 远程连接问题
端口检测
注册表：HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Terminal Server\WinStations\RDP-Tcp
查找PortNumber是不是3389，16进制就是d3d

火绒放行：
安全设置里面的网络防护，把对RDP、RDC相关的网络防护给关闭

系统问题
查看系统日志和版本，发现我的电脑是 Windows 11 专业版 24H2，操作系统版本 26100.4946。这个版本恰好受到微软 2026 年 1 月更新补丁的已知问题影响：安装特定累积更新后，远程桌面连接会失败，提示凭证错误或直接无法建立连接。微软随后发布了带外补丁 KB5077744 专门修复此问题。由于该补丁不会通过 Windows 自动更新推送，需要手动从微软更新目录网站下载。根据系统架构（x64）和版本（24H2），我选择了对应的补丁文件（windows11.0-kb5077744-x64_xxx.msu）进行安装，重启电脑后，远程桌面恢复正常。总结：此次无法连接的根本原因是系统更新补丁缺陷，而非火绒或防火墙设置问题。通过安装修复补丁即可解决。


我的解决路程
排查远程连接功能是否开启
排查远程端口是否是3389和是否放行
排查第三方防火墙拦截
排查系统问题

我最后的问题解决就是更新了系统才好，把火绒该开的网络防护都开了也能连接上

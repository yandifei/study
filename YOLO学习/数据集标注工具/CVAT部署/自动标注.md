# 核心应用场景分类

| 场景                      | 推荐方案                        | 核心优势                           | 局限性                       |
| ------------------------- | ------------------------------- | ---------------------------------- | ---------------------------- |
| **个人开发者 / 私有数据** | 本地 Docker 部署 + CLI / Nuclio | 数据不出域，完全免费，性能最强     | 需要配置本地环境             |
| **异地团队协作**          | 网页版 CVAT + SDK 脚本          | 无需部署服务器，直接处理云端任务   | 受限于网络上传速度           |
| **快速原型测试**          | 网页版 CVAT + 第三方集成        | 零代码，直接调用 Hugging Face 模型 | 存在调用额度限制或第三方费用 |

***

在使用 CVAT（Computer Vision Annotation Tool）对视频数据进行自动标注时，根据你是否拥有自己的训练模型，主要有三种实现路径。

以下是为您筛选的相关教程视频及其核心步骤总结：

## 二、 场景方案详解

### 1. 本地 Docker 部署 CVAT

如果你通过 Docker 自建了 CVAT 服务器，这是功能最完整的方案。

#### A. 显卡用户 (推荐)

* **实现方式：** 通过 **Nuclio** 部署 Serverless 函数。
* **配置要点：** * 安装 `nvidia-container-toolkit`。
* 在 `docker-compose.override.yml` 中开启 GPU 支持。


* **优点：** 可以在 CVAT 网页界面直接点击“自动标注”，推理速度极快。
* **费用：** **完全免费**。

#### B. 无显卡用户 (CPU 推理)

* **实现方式：** 同样通过 Nuclio 部署，但需选择 OpenVINO 优化后的模型或纯 CPU 版镜像。
* **现状：** 对于 YOLO 系列或 SAM 2 等大模型，CPU 推理延迟较高，建议只处理少量图片或短视频。

---

### 2. 非本地部署 (app.cvat.ai 网页版)

网页版无法直接访问你的本地硬件，因此自动标注逻辑有所不同。

#### A. 使用 CVAT SDK/CLI (开发者首选)

* **原理：** 在你本地电脑跑模型推理，通过 API 将结果“推”送到 CVAT 云端。
* **操作：** 1.  本地 Python 环境安装 `cvat-sdk`。
2.  运行推理脚本：`cvat-cli auto-annotate <TASK_ID> --function-file your_script.py`。
* **费用：** **免费**（仅消耗本地电力）。
* **硬件要求：** 取决于你本地电脑的配置。如果你有高性能显卡，这是最高效的办法。

#### B. 集成第三方平台 (Hugging Face / Roboflow)

* **原理：** CVAT 云端服务器直接对接这些平台的 API。
* **费用说明：**
* **CVAT 端：** 免费版账号通常有每月调用次数限制（如 100 次），超出需订阅付费版。
* **第三方端：** Hugging Face 的 Serverless API 基础版免费但有速率限制；若使用 **Inference Endpoints**（专用推理终端）则需按小时付费（GPU 实例通常 $0.5/h 起）。



---

## 三、 自动标注操作路径总结

### 路径 1：CVAT CLI (命令行)

**适用于：** 已有 `.pt` 或 `.h5` 模型，不想折腾 Docker 配置的开发者。

* **步骤：** 编写推理逻辑 -> 调用 `cvat-cli` -> 自动填充标注。
* **参考视频：** [CVAT.ai #21: Auto-Labelling via CLI](https://www.youtube.com/watch?v=T68fQJNHG84)

### 路径 2：Nuclio Serverless (网页端一键标注)

**适用于：** 本地部署，且需要团队成员在 UI 界面点击操作。

* **步骤：** 编写 `function.yaml` -> 使用 `nuctl` 部署到 Docker -> CVAT 自动识别模型。
* **参考视频：** [Semi and Full Automatic Annotation](https://www.youtube.com/watch?v=qYQE95VjfmA)

### 路径 3：云端集成

**适用于：** 模型已托管在 Hugging Face，或不具备本地硬件环境。

* **步骤：** 关联 API Key -> 选择模型 -> 标签映射 -> 开始标注。
* **参考视频：** [Accelerate with Hugging Face](https://www.youtube.com/watch?v=SBWv4Y9jkHI)

## 四、 性能与成本对比表

| 维度         | 本地 Docker (GPU)   | 网页版 + CLI (本地 GPU 推理) | 网页版 + 第三方平台     |
| ------------ | ------------------- | ---------------------------- | ----------------------- |
| **推理速度** | 极快 (原生支持)     | 极快                         | 取决于云端分配资源      |
| **费用**     | $0                  | $0                           | 可能产生订阅费或 API 费 |
| **配置难度** | 高 (需配置环境)     | 中 (需简单脚本)              | 低 (UI 绑定)            |
| **数据隐私** | 最高 (数据不离本地) | 中 (数据在云端，推理在本地)  | 低 (模型和数据都在云端) |

## 五、 实用建议

1. **如果你有显卡且不想付费：** 无论 CVAT 部署在哪，请务必学习使用 **CVAT CLI/SDK**。它能让你在本地显卡上跑完模型后，秒速同步标注到网页版。
2. **如果是视频任务：** 优先使用带有 **Tracker (追踪器)** 功能的模型，这能极大减少对视频中间帧的漏检修补工作。

### 1. 使用 CVAT CLI (命令行界面) 运行自定义模型（推荐，灵活性最高）

如果你已经有训练好的模型（如 YOLOv8, PyTorch 模型等），最快的方法是使用 CVAT 的 SDK 或 CLI 工具。这种方法不需要将模型部署到复杂的服务器环境中。

* **核心视频：** [CVAT.ai Product Tour #21: Use Any Model for Auto-Labelling via CLI](https://www.youtube.com/watch?v=T68fQJNHG84)
* **关键步骤：**
* 准备一个 Python 脚本来加载你的模型并定义推理逻辑。
* 使用 `cvat-cli` 命令行工具。
* 执行命令：`cvat-cli auto-annotate <TASK_ID> --function-file your_model_script.py`。
* **优点：** 无需配置 Nuclio，适合本地快速处理。



### 2. 通过 Nuclio 部署为 Serverless 函数（标准企业级做法）

如果你希望在 CVAT 的 Web 界面中直接点击“自动标注”按钮，你需要将模型封装成一个 Serverless 函数。

* **核心视频：** [Semi and Full Automatic Annotation | CVAT | Intel Software](https://www.youtube.com/watch?v=qYQE95VjfmA)
* **关键步骤：**
* 使用 **Nuclio** 框架将你的训练模型（如 OpenVINO, TensorFlow, PyTorch）构建为 Docker 镜像。
* 在 CVAT 的“Models”页面可以看到部署好的模型。
* 在视频任务中，点击 **AI Tools** -> **Detectors**，选择你的模型。
* 点击 **Annotate**，系统会自动为整段视频生成每一帧的标注。



### 3. 使用 AI Agent 或 集成第三方平台（如 Hugging Face/Roboflow）

如果你的模型托管在 Hugging Face 或 Roboflow 上，CVAT 提供了直接集成的接口。

* **相关视频：**
* [Accelerate annotation with Hugging Face](https://www.youtube.com/watch?v=SBWv4Y9jkHI)：介绍如何快速连接云端模型。
* [How to automatically annotate with AI agents (SAM2)](https://www.youtube.com/watch?v=E7Xw0jg7ZHE)：展示了使用最新的 AI Agent（如 SAM2）进行追踪和自动标注的过程。



### 总结建议
1. Nuclio Serverless,团队协作、长期项目,本地 Docker 部署,Nuclio + Docker,网页端一键调用，全团队共享
2. CVAT CLI / SDK,开发者自用、快速处理,本地 & 网页版均可,Python + CVAT API,灵活，无需配置复杂的服务器环境
3. 第三方平台集成,已有云端模型（HF/Roboflow）,网页版 (app.cvat.ai),API Key 授权,无需本地算力，配置简单

# 个人实践
文档参考：
https://docs.cvat.ai/docs/administration/community/advanced/installation_automatic_annotation/

https://docs.cvat.ai/docs/api_sdk/sdk/auto-annotation/

```bash
pip install cvat-sdk cvat-cli -U
```
### 一行命令开启“远程标注”
核心原理：远程 API 注入：你不需要下载图片。CVAT CLI 会通过 API 逐帧读取网页版任务中的图片，发送给你本地的显卡进行推理，然后实时把结果写回网页版。

如果使用的是CVAT内置的模型就可以不用py文件做适配，但是如果是自己的预训练模型就必须要做适配。

在终端执行以下命令，你会看到本地显卡开始疯狂运转，而网页版的标注在实时增加：

```bash
cvat-cli --server-host app.cvat.ai --auth <你的用户名>:<你的密码> \
auto-annotate <TASK_ID> \
--function-file ./yolo_wrapper.py \
--model-path ./your_model.pt \
--allow-unmatched-labels
```

* **`--server-host app.cvat.ai`**：连接远程网页版服务器。
* **`<TASK_ID>`**：你的网页任务 ID（浏览器地址栏里的数字）。
* **`--function-file`**：指向那个通用的适配脚本。
* **`--model-path`**：指向你本地的 `.pt` 模型文件。


CLI和SDK是不同的，CLI也要一个py文件但会传递模型位置参数，但是SDK就是直接运行一个py文件




在 CVAT 中导入 YOLO 格式的数据集，最简单且标准的方法是将图片和标签按照特定的文件夹结构打包成 **ZIP 压缩包**。

由于你使用的是 YOLO 目标检测格式（通常包含 `.txt` 标签文件和 `obj.names` 或 `dataset.yaml`），请参考以下步骤进行操作：

### 1. 准备文件夹结构

CVAT 对 YOLO 导入格式有严格要求，建议采用以下结构：

```text
dataset.zip
├── obj.names          (包含类别名称，每行一个)
├── obj.data           (定义路径和类别数量，见下文)
├── train.txt          (包含所有训练图片的相对路径)
└── obj_train_data/    (放所有图片和对应的 .txt 标签)
    ├── image1.jpg
    ├── image1.txt
    ├── image2.jpg
    ├── image2.txt
    └── ...

```

#### 关键文件内容示例：

* **obj.names**:
```text
person
car
dog

```


* **obj.data**:
```text
classes = 3
train = train.txt
names = obj.names

```


* **train.txt**:
```text
obj_train_data/image1.jpg
obj_train_data/image2.jpg

```



### 2. 执行导入步骤

1. **创建任务 (Create Task)**：
* 在 CVAT 控制台点击 `+` -> `Create a new task`。
* 填写任务名称。


2. **上传数据**：
* 在 **Constructor** 区域，不要直接拖入散乱的文件，而是点击 **"Connected file share"** 或 **"Local files"**。
* 选择你刚才打包好的 `dataset.zip` 文件。


3. **选择导入格式 (Important)**：
* 在页面下方的 **"Detailed configuration"** 中，找到 **"Import dataset from"** 选项。
* 在下拉菜单中选择 **`YOLO 1.1`**（这是 CVAT 兼容 YOLO 标签的标准选项）。


4. **提交**：
* 点击 `Submit & Open`，CVAT 会自动解析 ZIP 包并将标签关联到图片上。



---

### 3. 如果你已经创建了任务（只有图片，想补录标签）

如果你已经把图片上传到了任务中，现在只想把 `.txt` 标签导进去：

1. 进入该 **Task** 的详情页面。
2. 点击右上角的 **"Actions"** -> **"Upload annotations"**。
3. 在弹出的格式列表中选择 **`YOLO 1.1`**。
4. 上传包含上述结构（`obj.names`, `obj.data` 和所有 `.txt` 文件）的 ZIP 包。

### 常见问题排查：

* **图片路径不对**：确保 `train.txt` 里的路径能准确指向 `obj_train_data/` 目录下的图片。
* **类别不匹配**：如果导入后看不到框，检查 `obj.names` 里的类别顺序是否与你 YOLO 标签里 `class_id` 的索引一致。
* **多文件夹情况**：如果你有 `train` 和 `val` 两个文件夹，建议分别创建两个 CVAT Task 导入，或者合并到一个 `obj_train_data` 中通过 `train.txt` 管理。

此外，由于 CVAT 的版本更新，你也可以尝试直接使用 **YOLOv5/v8** 格式的 `dataset.yaml` 结构进行打包上传，CVAT 的较新版本通常也能识别这种标准的 Ultralytics 结构。

CVAT（Computer Vision Annotation Tool）截图，这些选项是用于定义**标签（Label）类型**和**属性（Attributes）**的。在计算机视觉标注中，这决定了你如何框选目标。

以下是图中常见的选项含义解释：

### 1. 标注形状类型 (Shape Types)

这些选项决定了标注员在界面上能用什么工具来标注这个标签：

* **Rectangle (矩形)**：最常用的选项。用于目标检测（Object Detection），画一个方框把物体框起来。
* **Polygon (多边形)**：用于实例分割（Instance Segmentation）。可以沿着物体的边缘打点，勾勒出不规则的形状（比如人体、弯曲的道路）。
* **Polyline (折线)**：用于标注线条类物体，比如车道线、电线或路缘。
* **Points (点)**：用于关键点检测（Keypoint Detection），比如人脸特征点、人体关节或复杂的物体轮廓点。
* **Cuboid (长方体/3D框)**：用于伪3D标注，给物体增加深度信息，常用于自动驾驶中的车辆标注。
* **Skeleton (骨架)**：用于人体姿态估计，是一组预定义好的连接点的组合。
* **Tag (分类标签)**：没有具体的形状，是对整张图片或某个特定区域进行分类说明（比如“晴天”、“室内”）。

### 2. 属性设置 (Attributes)

在定义标签时，你通常还会看到一些额外的配置项：

* **Attribute Name (属性名称)**：给这个标签添加额外描述。例如，标签是“车辆”，属性可以是“颜色”或“车牌号”。
* **Mutable (可变的)**：如果勾选，表示这个属性在视频标注的不同帧之间可以改变（比如车辆的“遮挡状态”）。
* **Input Type (输入类型)**：
* **Select (下拉框)**：从预设的几个选项里选一个（如：红、黄、蓝）。
* **Checkbox (复选框)**：布尔值，勾选表示“是”，不勾选表示“否”。
* **Text (文本框)**：手动输入文字信息。
* **Number (数字)**：输入数值。



### 总结

简单来说，如果你是要做一个普通的**目标检测**项目（画框），你应该选择 **Rectangle**；如果你需要更精细地**抠图**，则选择 **Polygon**。

**CVAT (Computer Vision Annotation Tool)** 中“创建新任务 (Create a new task)”的配置界面。CVAT 是一个非常流行

## 1. 基础配置 (Basic configuration) - 图 1

这是创建一个标注任务最基本的设置：

* **Name (名称):** 任务的名字。图中填写的是 `yysls训练集`。
* **Project (项目):** 任务所属的项目组。将任务关联到项目可以统一管理标签（Labels）。

* **Labels (标签):** 如果关联了项目，会默认使用项目的标签。
* **Select files (选择文件):** 上传待标注的数据。
* **My computer:** 从本地上传（支持图片、视频或压缩包）。
* **Connected file share / Remote sources / Cloud Storage:** 从服务器挂载目录、远程链接或云存储（如 AWS S3）导入数据。
## 2. 高级配置 (Advanced configuration) - 图 2

这些选项决定了数据的处理方式和任务的分配逻辑：

* **Sorting method (排序方式):** 图片在标注时的出现顺序。
* **Lexicographical:** 按文件名大小写字母顺序（默认）。
* **Natural:** 自然排序（例如 2.jpg 在 10.jpg 之前）。
* **Random:** 随机排序。

* **Prefer zip chunks:** 勾选后，系统会将数据切分为多个压缩包处理，有助于提升大任务的加载效率。
* **Image quality (图像质量):** 压缩比例。100% 为无损，70%（图中设置）可以减少存储占用并加快网页加载速度，但在细小物体标注时可能影响清晰度。
* **Overlap size (重叠大小):** 如果将一个任务切分成多个 Job，Job 之间重叠的帧数。用于确保跨分段物体的连续性。
* **Segment size (段大小):** 一个任务包含多少帧。
* **Start frame / Stop frame / Frame step:** 视频专用。指定从哪一帧开始、到哪一帧结束，以及抽帧步长（例如 Step 为 2 则每隔一帧取一张）。
* **Chunk size:** 数据分块的大小。
* **Consensus Replicas (共识副本):** 填写大于 0 的数字（如 3），表示同一个任务会分配给 3 个人标注，用于比对标注一致性。
* **Issue tracker:** 关联外部的任务管理系统（如 Jira 或 GitHub Issues）。

## 3. 质量控制 (Quality) - 图 3

用于自动化或人工的质量校验：

* **Validation mode (验证模式):**
* **None:** 不启用特定的验证模式。
* **Ground Truth:** 设置一些“真值”帧。标注人员标注这些帧后，系统会自动对比其准确率。
* **Honeypots (蜜罐):** 隐藏的测试帧。标注员不知道哪些是测试帧，系统通过这些帧的表现来评估标注员的水平。

**建议：** 如果你只是个人标注，保持默认设置即可。如果是视频抽帧标注，记得调整 **Frame step** 以减少冗余。


### 核心视频处理选项 (必看)

处理视频时，以下三个选项直接决定了你的工作量和标注精度：

* **Frame step (帧步长):** * **建议填写：** `5` 到 `30` 之间。
* **原因：** 视频通常是 30fps 或 60fps。如果你设为 `1`，每一帧都要标，工作量巨大且相邻帧画面几乎一样。设为 `10` 表示每 10 帧取一张图。如果是高速运动物体，设小一点；如果是监控等缓慢画面，可以设大一点。


* **Start frame / Stop frame (起始/结束帧):**
* **建议填写：** 留空（默认处理全段）或根据视频剪辑点填写。
* **场景：** 如果视频前 10 秒是黑屏或片头，可以通过这个跳过无效片段。


* **Image quality (图像质量):**
* **建议填写：** `85` 或 `95`。
* **原因：** 视频抽帧后的单帧质量通常比原生照片差。如果设为默认的 `70`，运动模糊加像素块会导致边缘很难看清。建议调高一点以保证标注精度。

### 任务分段与协作 (Segment & Overlap)

如果你的视频很长（比如 10 分钟），一个人很难一口气标完，这时需要合理分段：

* **Segment size (段大小):** * **建议填写：** `500` 到 `1000`。
* **逻辑：** 这里的数字是指**抽帧后的帧数**。如果你的视频抽完帧有 5000 张，设置 `1000` 就会自动帮你切分成 5 个作业（Job），你可以分给 5 个人同时标。

* **Overlap size (重叠大小):**
* **建议填写：** `5` 到 `10`。
* **逻辑：** 在两个作业的交界处保留几帧重叠。这对于**追踪（Tracking）**非常重要，能确保前一个标注员标的物体在下一个标注员那里能接得上。

### 其他进阶建议

* **Sorting method (排序方式):** 务必保持 **Lexicographical**。视频帧必须按时间顺序排列，绝对不能选 Random。
* **Use cache (使用缓存):** 务必**开启**。这能显著提升你在网页端来回切换帧时的流畅度。
* **Prefer zip chunks:** 建议**开启**。CVAT 会在后台把视频切成小块压缩包，这样你标注到后面时，浏览器不需要加载整个视频文件，不容易崩溃。

### 快速填写模版 (以 1 分钟 30fps 视频为例)

如果你想快速开始，可以参考这个配置：

| 选项              | 填写值 | 预期效果                                              |
| ----------------- | ------ | ----------------------------------------------------- |
| **Frame step**    | `10`   | 每秒钟标 3 张图，工作量适中                           |
| **Image quality** | `90`   | 画面清晰，方便看清细节                                |

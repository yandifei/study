## 1️⃣ JSON

### 1. 是什么 + 特点

* **JavaScript Object Notation**
* 结构非常清晰，几乎所有语言都支持
* 规则非常严格、简单
* **缺点：不能写注释（官方规范里）**

### 2. 典型应用场景

* Web API 前后端数据传输
* 配置文件（前端、Node、Electron、工具链）
* 浏览器本地存储（LocalStorage 数据常用 JSON 序列化）

### 3. 能存储的数据类型 / 数据结构

JSON 支持的数据类型：

* `string` 字符串
* `number` 数字（整数与浮点）
* `boolean` 布尔：`true` / `false`
* `null`
* `object`：键值对（无序字典）
* `array`：有序数组

所以本质上是由：**对象 + 数组 + 上面这些标量** 拼出来。

### 4. mini 入门教程

一个 JSON 示例：

```json
{
  "name": "Tom",
  "age": 20,
  "is_active": true,
  "skills": ["python", "js"],
  "profile": {
    "city": "Shanghai",
    "score": 99.5
  }
}
```

Python 解析：

```python
import json

with open("config.json", "r", encoding="utf-8") as f:
    data = json.load(f)

print(data["name"])      # Tom
print(data["skills"][0]) # python
```

## 2️⃣ YAML

### 1. 是什么 + 特点

* “YAML Ain’t Markup Language”
* 用**缩进表示层级结构**（非常像 Python）
* 配置文件可读性极强，能写注释、支持复杂结构
* **缩进只能用空格，禁止 Tab**

### 2. 场景 & 专用领域

* DevOps 世界：

  * Docker Compose: `docker-compose.yml`
  * Kubernetes: 各种 `deployment.yaml`
  * GitHub Actions: `.github/workflows/*.yml`
* 各种后端服务配置、微服务配置文件

### 3. 能存储的数据类型 / 数据结构

本质和 JSON 很像，但更强一点：

* 标量（scalar）

  * 字符串、数字、布尔、null、日期时间等
* 列表（sequence）
* 字典/映射（mapping）
* 高级玩法：

  * 多文档（`---` 分隔）
  * 锚点 & 引用（`&name` / `*name`）

典型结构：

```yaml
app:
  name: myapp
  debug: true
  version: 1.0

server:
  host: 127.0.0.1
  port: 8000
  tags:
    - api
    - demo
```

### 4. mini 入门教程（Python）

```bash
pip install pyyaml
```

```python
import yaml

with open("config.yaml", "r", encoding="utf-8") as f:
    data = yaml.safe_load(f)

print(data["app"]["name"])
print(data["server"]["tags"])
```

YAML → Python 映射：

* YAML 字典 → `dict`
* YAML 列表 → `list`
* `true/false` → `bool`
* `null` → `None`

## 3️⃣ TOML

### 1. 是什么 + 特点

* **Tom’s Obvious, Minimal Language**
* 想做到：**像 INI 一样简单，又比 INI 严谨，且支持更多类型**
* 现在是很多现代项目的**官方配置格式**

### 2. 典型 & 专用场景

* Python 项目：

  * `pyproject.toml`（项目依赖、构建配置）
* Rust 项目：

  * `Cargo.toml`（项目配置）
* 各种工具配置：

  * Black、Poetry 等 Python 工具
* 适合作为**应用配置/工具配置**：结构清晰 + 支持注释

### 3. 能存储的数据类型 / 数据结构

支持的类型比 JSON 丰富一些：

* 字符串、整数、浮点、布尔
* 日期时间（真正的 datetime 类型）
* 数组（`[]`）
* 表（table）= 类似 dict
* 数组表（array of tables）

示例：

```toml
title = "My App"
debug = true
pi = 3.1415

[server]
host = "127.0.0.1"
port = 8000

[author]
name = "Tom"
email = "tom@example.com"

[features]
enabled = ["auth", "cache"]
```

### 4. mini 入门教程（Python 3.11+）

Python 3.11 内置 `tomllib`：

```python
import tomllib

with open("config.toml", "rb") as f:
    data = tomllib.load(f)

print(data["title"])
print(data["server"]["port"])
```

3.10 及以下：

```bash
pip install tomli
```

使用方式几乎一样。

## 4️⃣ INI

### 1. 是什么 + 特点

* 非常古老、经典的配置格式
* 常见于 Windows 软件、早期 PC 应用
* **语法简单，但类型支持非常弱**：所有值都是字符串，除非你手动转

### 2. 典型场景

* Windows 桌面软件配置
* 早期游戏 / 工具配置
* 一些简单脚本 / 服务的小配置

### 3. 能存储的数据结构

INI 非常“原始”：

* 顶层结构：**节（section） + key=value**
* 不支持嵌套对象 / 列表的标准语法（有些项目会自己发明约定）

示例：

```ini
[ui]
theme = dark
font_size = 14

[server]
host = 127.0.0.1
port = 8000
```

所有值其实都是字符串："14"、"8000"。

### 4. mini 入门教程（Python）

```python
import configparser

config = configparser.ConfigParser()
config.read("config.ini", encoding="utf-8")

theme = config["ui"]["theme"]
font_size = int(config["ui"]["font_size"])
port = int(config["server"]["port"])
```

你需要自己做类型转换。

## 5️⃣ XML

### 1. 是什么 + 特点

* **Extensible Markup Language**
* 使用标签 `<tag>...</tag>` 表示结构，类似 HTML
* 超级通用：可以描述文档、配置、消息……
* 缺点：**啰嗦、冗长，写起来累，读起来也累**

### 2. 典型 / 专用场景

现在 XML 更多出现在一些“老领域”或“标准领域”：

* Android 布局文件：`activity_main.xml`
* Office 文件底层格式（docx / xlsx 其实是一堆 XML 打包）
* 一些标准协议：SOAP、RSS 等
* 某些企业级老系统配置文件、Spring 旧时代的配置（Spring XML）

### 3. 能存储的数据结构

XML 本质上是：

* 元素（element）= 标签 + 子元素 + 属性
* 属性（attribute）= 键值对挂在标签上
* 文本（text）

你可以用它表达任何结构，但需要自己定义标签含义，如：

```xml
<person>
  <name>Tom</name>
  <age>20</age>
  <skills>
    <skill>python</skill>
    <skill>java</skill>
  </skills>
</person>
```

对应的逻辑结构类似：

```python
{
  "person": {
    "name": "Tom",
    "age": 20,
    "skills": ["python", "java"]
  }
}
```

但这个“映射过程”需要你自己设计或用库帮你做。

### 4. mini 入门教程（Python）

Python 内置 XML 解析库：

```python
import xml.etree.ElementTree as ET

tree = ET.parse("person.xml")
root = tree.getroot()

name = root.find("name").text
age = int(root.find("age").text)

skills = [e.text for e in root.find("skills").findall("skill")]
```

## 🔚 最后帮你总结：实战选型建议

如果你现在要 **设计/阅读 配置文件**，可以这么选：

* ✅ **JSON**

  * 场景：前后端通信、前端项目、简单配置
  * 优点：通用、工具多；缺点：不能写注释

* ✅ **YAML**

  * 场景：服务端配置、DevOps（Docker、K8s、CI/CD）
  * 优点：可读性强，支持复杂结构；缺点：语法灵活，容易写错，禁止 Tab

* ✅ **TOML**

  * 场景：现代项目配置（Python、Rust、工具链）
  * 优点：比 INI 规范，比 YAML 简单，支持注释 + 多类型

* ✅ **INI**

  * 场景：简单桌面应用、传统 Windows 世界
  * 优点：超简单，库多；缺点：类型弱，结构不够表达复杂配置

* ✅ **XML**

  * 场景：老系统、标准协议、Android、Office、企业遗留项目
  * 优点：生态广泛、可验证（XSD）；缺点：冗长，写起来累

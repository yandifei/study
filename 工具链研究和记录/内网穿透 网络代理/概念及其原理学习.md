## 一、概念定义（关键术语速查）

| 术语                     | 定义简述                                                                                             |
| :----------------------- | :--------------------------------------------------------------------------------------------------- |
| **反向代理**             | 公网服务器**主动**访问内网服务器，接收客户端请求后转发给内网的一台或多台服务器。                     |
| **正向代理**             | 内网客户端请求公网代理服务器，由代理**代为访问**目标资源并返回结果，服务端感知到的客户端是代理。     |
| **隧道穿透**             | 内网设备**主动向公网服务器发起连接**，建立一个持久、双向的通信隧道，以此打通内外网。                 |
| **反向隧道（入站隧道）** | 隧道的一种应用模式，将外网请求**引入内网**，解决无公网IP让外网访问内网服务的问题。                   |
| **出站隧道（正向隧道）** | 隧道的一种应用模式，将内网流量**导出公网**，用于代理上网、访问外网资源。                             |
| **端口映射**             | 在路由器上设置规则，将公网IP的特定端口流量转发到内网指定设备的端口，常配合 UPnP / NAT-PMP 自动实现。 |

---

## 二、原理解析（核心机制）

### 1. 反向代理（以 Nginx / Caddy 为例）
- **通讯模型**：请求-响应，支持 HTTP/2 多路复用。
- **工作流**：公网服务器部署反向代理 → 用户请求到达公网 → 代理根据规则 **主动向内网后端发起连接** → 返回响应。
- **必要条件**：内网后端必须在路由器上暴露端口映射，否则代理无法主动连入。
- **核心价值**：流量调度、SSL终结、缓存、安全防护。

### 2. 隧道穿透（以 frp / Cloudflare Tunnel 为例）
- **通讯模型**：全双工长连接，天然支持多路复用和高并发转发。
- **工作流（反向隧道）**：内网客户端主动向公网服务器建立加密隧道 → 并注册服务映射关系 → 外部访问公网服务器端口 → 流量沿隧道透明流入内网服务。
- **绝对优势**：**无需任何端口映射或公网IP**，即可将内网服务安全暴露到公网。
- **工作流（出站隧道/代理）**：内网客户端同样主动连接公网节点，但流量方向相反，用于本机访问互联网。

## 三、区别分析（对比表格）

| 对比维度             | 反向代理                     | 正向/出站代理          | 反向隧道穿透                |
| :------------------- | :--------------------------- | :--------------------- | :-------------------------- |
| **谁主动发起连接**   | 公网服务器（到内网）         | 内网客户端（到公网）   | 内网客户端（到公网服务器）  |
| **是否需要端口映射** | **必须**                     | 不需要                 | **不需要**                  |
| **流量最终方向**     | 外网 → 内网                  | 内网 → 外网            | 外网 → 内网                 |
| **连接模式**         | 短连接/请求-响应（可复用）   | **全双工加密长连接**   | **全双工加密长连接**        |
| **主要用途**         | 发布网站、负载均衡、统一入口 | 科学上网、访问受限资源 | 远程桌面、内网站点、家庭NAS |

> **一句话区分**：  
> - **反向代理** = 公网“叫门”，内网必须“开门”（端口映射）。  
> - **反向隧道** = 内网主动“挖地道”出去，公网顺着地道进来，无需开门。  
> - **出站代理/隧道** = 内网主动“挖地道”出去，通过隧道访问公网资源。

## 四、作用与代表软件（2026 主流版）

### 1. 反向代理类

| 软件                            | 简介与定位                                                                    |
| :------------------------------ | :---------------------------------------------------------------------------- |
| **Nginx / Nginx Proxy Manager** | 经典高性能反向代理，NPM 提供可视化配置面板，家庭服务器入门首选。              |
| **Caddy**                       | 自动申请和续期 HTTPS 证书，配置文件极为简洁，个人和小团队首选。               |
| **Traefik**                     | 为 Docker、Kubernetes 等动态环境设计，可自动发现服务并配置路由。              |
| **Pangolin**                    | 新兴家庭实验室方案，基于 Traefik + WireGuard 实现安全反向代理，无需开放端口。 |

### 2. 隧道穿透类（内网穿透）

| 软件                     | 类型       | 简介与定位                                                                     |
| :----------------------- | :--------- | :----------------------------------------------------------------------------- |
| **frp**                  | 自建型     | 开源标杆，功能全面，适合有一台公网 VPS 的用户自行搭建。                        |
| **NPS**                  | 自建型     | 带 Web 管理面板的穿透服务端，配置更直观，适合习惯图形化操作的用户。            |
| **Cloudflare Tunnel**    | 公共服务型 | 无限流量，免费全球加速，零配置 HTTPS，目前最流行的“白嫖”方案。                 |
| **Sakura Frp**           | 公共服务型 | 国内稳定、节点多、流量政策宽松的付费穿透服务，适合不想自建服务器的用户。       |
| **Tailscale / ZeroTier** | 组网型     | 将设备组成虚拟局域网，通过私有 IP 直接访问，相当于把“端口暴露”变为“内网互通”。 |
| **Ngrok**                | 公共服务型 | 开发者快速调试利器，一条命令生成公网链接，极其适合临时演示和测试。             |

## 🎯 终极总结

- **让外网访问内网服务（无公网IP）**：首选 **Cloudflare Tunnel**（免费）或 **frp/NPS**（自建），它们都是通过内网主动发起的反向隧道实现，无需端口映射。
- **搭建网站或统一入口（有公网IP）**：首选 **Nginx Proxy Manager** 或 **Caddy** 作为反向代理，配合端口映射使用。
- **内网设备安全访问外网**：使用 **Clash Verge Rev** 等代理客户端，本质也是建立一条出站隧道。

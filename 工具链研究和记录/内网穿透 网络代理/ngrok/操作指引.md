官网：
https://ngrok.com/

文档：
https://ngrok.com/docs/start


去到官网后进行进行账号注册，我这边遇到的情况是说验证失败，估计要特殊网络环境

之后它会分配1个永久域名来用

去docker-compose.yaml文件中修改ngrok的地址和填写`NGROK_AUTHTOKEN`

启动指令
```bash
docker compose up -d
```

**必须去访问自己的网址并点击vite**

# 我的修改点
官方用的是一条指令，我这边是docker compose实现的，可控都度更高，但是缺点也来了
我得删掉这个内容，并用`"host.docker.internal:80"`去代替原来的80端口
```
# 使用 host 网络模式，容器直接共享宿主机网络栈
# 效果等同 --net=host，此时容器可直接访问宿主机的 localhost 端口
network_mode: host
```

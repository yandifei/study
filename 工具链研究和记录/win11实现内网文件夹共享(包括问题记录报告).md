# 🪟 Windows 11 局域网共享免密访问问题排查与解决记录

> **适用场景**：Windows 11/10 主机开启文件夹共享，局域网内另一台电脑（笔记本）访问时，始终提示需要输入网络凭据。即使共享权限已设置为 `Everyone` 完全控制，且系统设置中已关闭“密码保护共享”，问题依然存在。

## 📍 问题现象
1. 主机共享文件夹的“共享”权限和“安全”权限均已赋予 `Everyone` 完全控制。
2. 网络属性已配置为“专用网络”，且开启了网络发现和文件共享，关闭了密码保护共享。
3. 访问端（笔记本）尝试连接 `\\主机名\共享文件夹`。
4. **现象 A**：系统依然强制弹出“输入网络凭据”的登录框。
5. **现象 B**：输入主机登录的**微软账号**邮箱和密码后，系统提示**“用户名或密码不正确”**（其实密码是正确的，但被 Win11 安全机制拦截）。

## 🛠️ 核心解决步骤（免密访问）

要彻底解决 Win11 的局域网共享问题，需要打通底层的安全策略。请按以下步骤在**主机（作为服务端）**操作：

### 步骤一：检查组策略中的 Guest 来宾账户
Win11 默认禁用了 `Guest` 账号，导致即使共享权限开放，外部设备也无法直接进入。
*   **方法 A（适用于专业版/企业版）：** 打开 `gpedit.msc` -> 计算机配置 -> Windows 设置 -> 安全设置 -> 本地策略 -> 安全选项，找到 **“账户：来宾账户状态”**，设置为 **“已启用”**。
*   **方法 B（适用于家庭版）：** 按 `Win + R` 输入 `cmd`，执行命令 `net user guest /active:yes`。

### 步骤二：修改底层网络身份验证策略（必做）
这是 Win11 顽固的安全防御机制（默认拒绝不安全的来宾登录）。
*   **方法 A（组策略）：** 在上一步的“安全选项”里，找到 **“网络安全：LAN Manager 身份验证级别”**，修改为 **“发送 LM 和 NTLM 响应”**（或 `发送 LM 和 NTLM - 如果协商，则使用 NTLMv2 会话安全`）。
*   **方法 B（注册表，推荐家庭版用）：** 打开 `regedit`，定位到 `HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Lsa`。新建一个 DWORD（32位）值，命名为 `AllowInsecureGuestAuth`，并将其值改为 `1`。

### 步骤三：清理访问端的“错误缓存”（关键）
如果访问端**以前输入过错误的密码**，Windows 会将错误的密码“记仇”在凭据管理器中。
*   **操作方法**：在**笔记本（访问端）**上打开“控制面板” -> “用户账户” -> “凭据管理器” -> “Windows 凭据”。在“普通凭据”中找到主机名相关的项，**删除**它们。

### 步骤四：重启主机电脑（最后一步！）
**这一步极其重要！** 所有组策略、注册表、以及开启 Guest 账户的操作，必须**重启主机**才会彻底生效并在底层重新加载安全策略。

---

## ⚠️ 避坑指南（关于提示输入密码）

如果你按照以上步骤操作完重启后，**系统仍然提示输入密码**（这种情况偶尔会发生，属于正常现象），请务必使用以下**正确的登录格式**：

*   ❌ **错误做法**：直接在用户名处输入主机的 Microsoft 邮箱账号密码（会大概率报错）。
*   ✅ **正确做法**：在登录框的“用户名”处输入 `主机名\本地账户名`。例如 `Yandifei\Admin`（其中 `Admin` 是主机上本地管理员或新建的本地测试账户名）。**密码**则输入该**本地账户的开机登录密码**。

*(此方法可以通过“凭据管理器”保存一次凭证，之后每次连接即可自动秒进，无需反复输入。)*

## ✅ 最终结论
Windows 11 的网络共享机制比老系统多了两层安全锁：
1. **Guest 账号是否启用**（决定外设能不能进大门）。
2. **LM 身份验证级别**（决定同不同意不加密的来宾访问）。
配合 `Everyone` 权限设置、网络发现开启以及清空缓存，就能完全实现局域网内方便的免密共享。

# 挂载
nohup：让命令忽略你注销/断开的挂断信号，你关了 SSH 窗口服务还在。
> /dev/null 2>&1：把所有的输出都丢掉，不产生日志文件。
最后的 &：扔到后台运行。
执行后，屏幕会显示类似 [1] 12345，这个 12345 就是进程 PID（编号），你可以记下来。这时你就可以安全地关闭终端窗口了，服务依然在运行。

进入资源所在目录后使用以下指令
nohup python3 -m http.server 60000 > /dev/null 2>&1 &


按端口查看是否使用
netstat -tlnp | grep 8080

关闭这个服务
kill PID

---

### 关于 `python3 -m http.server 8080` 的总结

#### 1. 命令含义
- 这是一个**临时、轻量级的只读 Web 文件服务器**，使用 Python 自带的模块。
- `8080` 是服务监听的端口号，可以自行修改。
- 它会将**执行命令时所在的文件夹**作为网站根目录，并将其中的所有文件提供给浏览器访问。

#### 2. 路径共享方式
- 需要先 `cd` 到目标文件夹（如 `/home/yandifei`），再执行该命令。
- 共享的不是单独文件，而是**一整个目录**。链接格式为 `http://你的IP:8080/文件名`。
- 例如：视频 `homework.mp4` 就在 `/home/yandifei/` 下，则访问链接为 `http://IP:8080/homework.mp4`。

#### 3. 安全性分析
- **只读**：服务器不支持上传、修改或删除操作，无法被植入病毒。
- **无法上层访问**：不能通过 `../` 等路径跳出当前目录去访问系统文件（如 `/root`）。
- **低爆破风险**：因为只能读取已有文件，只要目录里没有敏感文件，就不会造成信息泄露。
- **建议做法**：为避免意外暴露其他文件，可创建一个临时目录，只放需要分享的文件，再启动服务。
  ```bash
  mkdir ~/tmp_share
  cp ~/homework.mp4 ~/tmp_share/
  cd ~/tmp_share
  python3 -m http.server 8080
  ```
- **用完即关**：老师下载完成后，按 `Ctrl+C` 停止服务，端口立即关闭，风险消失。

1. 进到文件所在的文件夹，再跑命令。
2. 这是只读的，不能传毒，也翻不到系统盘。
3. 用完立刻关掉，安全无残留。

### **问题一：为什么C盘有一个4GB的 `pip` 文件夹？我一直用Conda，不应该有这个啊？**

**我的回答核心：**
1. **原因**：这个文件夹是 `pip` 的**下载缓存**（位于 `C:\Users\用户名\AppData\Local\pip\cache`）。当你激活Conda环境后执行 `pip install` 时，`pip` 会默认把下载的安装包缓存到系统盘，导致体积膨胀到4GB。
2. **安全性**：删除该文件夹**不影响已安装的库**，只会在下次重新安装时重新下载。
3. **清理方法**：运行 `pip cache purge` 直接清空缓存。
4. **预防建议**：
   - 转移缓存目录（配置 `pip.ini` 到其他盘）
   - 定期执行 `pip cache purge`
   - 尽量在一个环境中统一使用 Conda 或 Pip，减少混用

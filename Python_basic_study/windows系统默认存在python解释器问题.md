## 📋 问题回顾与解决方案总结

### 1. 你的初始提问

**现象**：在命令行输入 `where python` 发现两个路径：

* `A:\Anaconda\python.exe`（你真正想要的）
* `C:\Users\yandifei\AppData\Local\Microsoft\WindowsApps\python.exe`（碍事的）

**诉求**：想删除 Conda 以外的解释器，甚至想直接删除 `WindowsApps` 文件夹。

### 2. 最终解决方法（避坑指南）

**不能做的事**：

* ❌ **绝对不能**删除整个 `WindowsApps` 文件夹，因为它包含记事本、画图和终端等系统核心组件。
* ❌ **不建议**手动删除文件夹里的 `python.exe`，因为它是系统生成的“别名”。

**正确的操作步骤**：

1. **路径**：进入 `设置` -> `应用` -> `高级应用设置` -> `应用执行别名`。
2. **操作**：在列表中找到名为 **“应用安装程序 (python.exe)”** 和 **“应用安装程序 (python3.exe)”** 的选项。
3. **关闭**：将这两个开关拨到 **“关”**。
4. **验证**：重启命令行，输入 `where python`，此时应仅显示 Anaconda 的路径。

## 🧐 深度解析：为什么会多出一个 Python？

### 为什么会产生这个“多余”的 Python？

这其实是微软为了提高“用户友好度”设计的 **“诱饵” (Alias)**。

* 当你在一台全新的 Windows 上输入 `python` 时，如果系统发现你没装 Python，它不想让你看到“命令不存在”的报错，而是希望引导你去 **Microsoft Store（微软商店）** 下载。
* 所以它在 `WindowsApps` 目录下预放了两个 **0 KB** 的“钩子”文件。你运行它，它就带你去商店；你装了 Anaconda，它就变成了你路径里的“拦路虎”。

### 这是从哪个版本开始的？

这个功能是从 **Windows 10 版本 1903 (May 2019 Update)** 开始引入的。

### 为什么以前没有这回事？

1. **策略转变**：早期 Windows 认为 Python 是专业开发者的工具，不关系统的事。但随着 Python 变得极其流行，微软决定将其引入应用商店，并像自带记事本一样提供“快速入口”。
2. **UWP 架构**：Windows 10 推广 UWP（通用 Windows 平台）应用，`WindowsApps` 目录就是这些应用的存放地。为了让这些应用能从命令行启动，微软发明了“应用执行别名”机制。

# 实现换行的Unicode字符
| 名称               | 代码点              | 转义序列       | 说明                                        |
| :----------------- | :------------------ | :------------- | :------------------------------------------ |
| 换行符 (LF)        | `U+000A`            | `\u000A`       | Unix/Linux/macOS 标准换行符                 |
| 回车符 (CR)        | `U+000D`            | `\u000D`       | 移动光标到行首，与 LF 组合成 Windows 换行符 |
| 回车换行符 (CR+LF) | `U+000D` + `U+000A` | `\u000D\u000A` | Windows 系统换行符（两个字符组合）          |
| 行分隔符 (LS)      | `U+2028`            | `\u2028`       | Unicode 显式行分隔符，类似纯文本的 `<br>`   |
| 段落分隔符 (PS)    | `U+2029`            | `\u2029`       | Unicode 显式段落分隔符，表示段落结束        |
| 下一行 (NEL)       | `U+0085`            | `\u0085`       | 兼容 EBCDIC 系统的换行控制，效果近似 CR+LF  |
| 换页符 (FF)        | `U+000C`            | `\u000C`       | 打印机换页，文本中极少使用                  |
| 垂直制表符 (VT)    | `U+000B`            | `\u000B`       | 垂直方向制表，现已罕见                      |
| 零宽度空格 (ZWSP)  | `U+200B`            | `\u200B`       | 不可见，用于指示单词内可换行位置            |

如果你需要在编程语言中使用这些转义，注意不同语言对 Unicode 转义的支持：
- JavaScript/JSON: 使用 `\uXXXX` 形式，如 `\u0085`。
- Python: 同样支持 `\uXXXX`，但需放在字符串中。
- Java/C#: 也使用 `\uXXXX`。
- HTML: 使用 `&#xXXXX;` 或 `&#dddd;`，如 `` 表示 U+0085。
- 直接输入: 部分操作系统支持通过输入法或 Alt 码输入。

# 信息隐藏
将一个中文字符转为Unicode字符
然后对每个Unicode字符用零宽字符隐藏起来
## 原理展示
以变体选择器为例（假设使用16个扩展变体选择器U+E0100 ~ U+E01EF，每个可以表示一个十六进制数字）：

将“喵”的Unicode码点（U+55B5）转换为十六进制字符串：55B5。

将每个十六进制字符映射到一个变体选择器：例如 5 → U+E0105，5 → U+E0105，B → U+E010B，5 → U+E0105。

这样，秘密信息就被编码成了 [U+E0105, U+E0105, U+E010B, U+E0105] 四个不可见字符。

其他编码方式：

用两个零宽字符代表二进制（如U+200B表示0，U+200C表示1），将“喵”的二进制形式嵌入。

直接使用Unicode的私有区域字符（但兼容性较差）。

## 方法
零宽空格本身不可见。
当别人@你并输入内容时，他们的输入会跟在昵称后面，而昵称后面是一串零宽空格。


# QQ后缀实现
为了实现让群友每次@我时都变成猫娘喊一声“喵~”的效果，标题中的“后缀”即指昵称在显示时会有部分后置。根据前人经验，我们只需要合理使用RLO（从右到左强制）和LRO（从左到右强制）控制字符即可实现。而实操过程中，发现使用
（部分昵称）＋RLO＋（部分昵称）＋LRO
这个格式，将无法修改成功。查询相关控制字符后，尝试使用
（部分昵称）＋（从右到左隔离符号）＋（部分昵称）＋（从左到右隔离符号）
即可成功修改并实现所需效果。

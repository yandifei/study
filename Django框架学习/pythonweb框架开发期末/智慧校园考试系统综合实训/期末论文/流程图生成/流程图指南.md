### 📝 核心绘制规范
为了确保流程图清晰、规范，文章强调了以下关键规则：
*   **图形与流向**：使用**统一的形状和大小**；流程方向默认应**从左到右、从上到下**；连接线尽量避免交叉。
*   **开始与结束**：每个流程**必须只有一个开始符**（如椭圆），但**可以有多个结束符**。
*   **判断框的使用**：判断框（菱形）的**上下端通常连接“是”分支**，**左右端连接“否”分支**。
*   **流程说明**：完整的流程图应附带说明，包括流程目的、适用范围、职责分工和关键节点说明。

### 🔵 基本图形元素
绘制流程图需使用标准符号，其常见含义如下：
| 图形           | 名称      | 含义                           |
| :------------- | :-------- | :----------------------------- |
| **椭圆形**     | 开始/结束 | 表示流程的起点或终点。         |
| **矩形**       | 处理过程  | 表示一个具体的操作或活动。     |
| **菱形**       | 判断      | 表示决策或审核点，会产生分支。 |
| **平行四边形** | 输入/输出 | 表示数据的输入或输出。         |
| **箭头**       | 流程线    | 表示步骤的顺序和流向。         |

### 📋 设计流程图的基本步骤
无论是梳理业务还是设计测试用例，都可以遵循以下四个步骤：
1.  **了解需求**：详细理解业务流程或程序逻辑。
2.  **梳理关系**：找出流程中涉及的各个环节（如页面、模块）及其之间的流转顺序。
3.  **绘制图形**：根据梳理出的逻辑，使用规范符号画出流程图。
4.  **覆盖路径**：基于流程图，检查并确保所有可能的路径分支都已被考虑（对于测试，即设计用例覆盖所有分支）。

### 🔁 两种循环结构的画法
文章特别用Python代码示例说明了如何绘制两种常见的循环结构流程图：
*   **For循环**：流程图核心包括**初始化**、**循环条件判断**（菱形框）、**执行循环体**（矩形框）和**更新循环变量**的步骤。
*   **While循环**：首先进行**条件判断**（菱形框），如果条件为“是”，则进入**循环体**执行操作，执行完后**返回再次判断条件**，形成循环。

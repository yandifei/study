#### 动态加载美化
我发现这这个加载进入界面后`文化传承热度`这个进度条是静态的，我想要给他搞成动态，就是加载进度条会不断加载，ok直接上代码

onLoad() {
  // 显示 loading，模拟加载过程
  // 省略代码.....
  // 启动进度条，接上计时器的异步
  this.startProgressAnimation(100);
  // 省略后面的代码
},

// 进度条推进
startProgressAnimation(target) {
  let current = this.data.heat;
  const step = () => {
    if (current < target) {
      current++;
      this.setData({ heat: current });
      setTimeout(step, 20);  // 每 20ms 增加 1，2 秒左右到 100
    }
  };
  step();
}

### 2. 文化测验页（quiz）
**主要功能**：以表单形式进行广东文化知识测验，包含多种表单组件，支持提交计分与重置。

- **单选题**：询问广东传统戏剧（radio 组件）。

- **判断题**：询问粤语是否为广东主要方言（switch 组件）。
- **评分滑块**：对广东文化热爱程度打分（slider 组件）。
- **文本输入**：填写最喜欢的广东美食（input 组件）。
- **下拉选择器**：选择最喜欢的广东城市（picker 组件）。
- **提交与重置**：点击提交计算得分并弹出模态框，重置清空表单并提示。

## 四、心得体会（遇到的困难及解决方法）
- **轮播图图片路径问题**  
  这个路径给我整懵逼了，之前我记得是无论身处何处使用相对路径，这个相对路径是Page下开始的相对，现在都变成了这个页面文件的相对路径，应该和我最近的一次更新有关。
  **解决方法**：使用 `/Page/images` 这种相对当前项目目录下的绝对路径，并确保图片存放于项目根目录的 images 文件夹里面。

- **多选评分逻辑处理**  
  文档要求多选题完全匹配才得分，未修改的版本使用 `include` 判断不严谨。  
  **解决方法**：将正确答案（`['xiajiao','changfen','shaomai']`）和用户提交的选项数组分别排序后转为字符串比较，确保顺序不影响匹配。

- **switch 组件取值**  
  表单提交时 `e.detail.value` 中 switch 的值为 `true`/`false` 而非直接字符串，导致计分判断出错。  
  **解决方法**：在计算时使用 `=== true` 判断开关状态。

- **slider 与 progress 联动**  
  初始时 progress 不随 slider 实时更新。  
  **解决方法**：在 slider 的 `bindchange` 事件中通过 `setData` 同步更新绑定变量。
  **二次修bug：**
  后面发现我不是滑动中进度条就会改变而是停止滑动后才会改变，然后发现用错了绑定事件，应该用`bindchanging`才对。

- **页面跳转问题**
  首页点击按钮没反应，没给我跳到文化测试界面
  **解决方法**：
  跳转 Tab 页须使`open-type="switchTab"`。跳转普通页面使用 `open-type="navigateTo"`。

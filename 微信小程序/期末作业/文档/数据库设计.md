# 数据库设计文档

MongoDB 物理数据模型。采用嵌入式文档模式，浏览和收藏不建独立集合。

---

## 一、集合设计

### 1.1 users（用户）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `_id` | ObjectId | PK | 代理主键 |
| `email` | String | UNIQUE | 登录凭证 |
| `username` | String | — | 用户昵称 |
| `browse_records` | Array\<BrowseItem\> | 最多 1000 条 | 浏览记录（嵌入式，按 image_id 降序，新→旧） |
| `favorite_records` | Array\<FavoriteItem\> | 无上限 | 收藏记录（嵌入式，按 image_id 降序，新→旧） |
| `created_at` | DateTime | — | 创建时间 |
| `updated_at` | DateTime | — | 更新时间 |

**BrowseItem 子文档：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `image_id` | String | 图片唯一标识（时间戳） |
| `image_url` | String | 图片 URL（冗余） |
| `browse_time` | DateTime | 浏览时间 |

**FavoriteItem 子文档：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `image_id` | String | 图片唯一标识（时间戳） |
| `image_url` | String | 图片 URL（冗余） |
| `favorite_time` | DateTime | 收藏时间 |

索引：`email` 唯一索引，`username` 普通索引

```json
{
  "_id": ObjectId("..."),
  "email": "admin@qq.com",
  "username": "admin",
  "browse_records": [
    { "image_id": "20250610120000", "image_url": "https://cdn.example.com/a.jpg", "browse_time": ISODate }
  ],
  "favorite_records": [
    { "image_id": "20250610120000", "image_url": "https://cdn.example.com/a.jpg", "favorite_time": ISODate }
  ],
  "created_at": ISODate("2025-01-01T00:00:00Z"),
  "updated_at": ISODate("2025-01-01T00:00:00Z")
}
```

### 1.2 images（图片）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `_id` | String | PK | 时间戳作为主键（如 `20250610120000`） |
| `url` | String | UNIQUE | 图片资源地址（OSS/CDN） |
| `type` | String | — | 图片格式（jpg, png, webp, gif） |
| `created_at` | DateTime | — | 入库时间 |

索引：`url` 唯一索引，`type` 普通索引，`created_at` 降序索引

```json
{
  "_id": "20250610120000",
  "url": "https://cdn.example.com/photo.jpg",
  "type": "jpg",
  "created_at": ISODate("2025-06-10T12:00:00Z")
}
```

## 二、E-R 模型（概念层）

### 2.1 实体

| 实体 | 属性 | 主键 |
|------|------|------|
| **用户 (User)** | id, 邮箱, 用户名 | id |
| **图片 (Image)** | id（时间戳）, URL, 类型 | id |

### 2.2 关系

| 关系 | 关联 | 基数 | 物理实现 |
|------|------|------|---------|
| **浏览 (Browse)** | 用户 ↔ 图片 | m:n | 嵌入 User.browse_records，最多 1000 条 |
| **收藏 (Favorite)** | 用户 ↔ 图片 | m:n | 嵌入 User.favorite_records，无上限 |

### 2.3 基数

| 实体 | 关系 | 基数 | 含义 |
|------|------|------|------|
| 用户 | 浏览 | m | 一个用户可浏览多张图片 |
| 图片 | 浏览 | n | 一张图片可被多个用户浏览 |
| 用户 | 收藏 | m | 一个用户可收藏多张图片 |
| 图片 | 收藏 | n | 一张图片可被多个用户收藏 |

## 三、设计说明

1. **嵌入式数组**：浏览和收藏记录嵌入在 `users` 文档中，无需 JOIN。MongoDB 16MB 文档上限下，1000 条浏览 + 收藏完全够用。

2. **浏览上限 1000**：使用 `$push` + `$slice: -1000` 自动裁剪旧数据，按 `image_id` 降序排列（新→旧）。

3. **收藏去重**：添加收藏时先 `$pull`（删除同 image_id 旧记录）再 `$push` + `$sort`（降序），天然去重。

4. **图片时间戳 ID**：图片以时间戳（如 `20250610120000`）作为主键，利于排序和追溯。

5. **冗余 `image_url`**：浏览/收藏子文档中冗余存 URL，查询历史列表时无需再查 `images` 集合。

6. **分页查询**：使用 MongoDB `$slice: [skip, limit]` 在服务端做分页，不会把全部数据拉回应用层。接口层 `limit` 硬上限 100，翻页靠 `skip` 递增。

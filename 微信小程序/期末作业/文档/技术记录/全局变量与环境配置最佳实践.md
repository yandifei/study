# 微信小程序全局变量与环境配置最佳实践（个人上线版）

## 一、核心痛点
- URL 地址经常变，分散在多个 `.js` 文件里难以维护。
- 开发、测试、正式环境地址不同，手动切换容易出错。
- 需要跨页面共享数据（如用户登录态），但不想引入复杂框架。

## 二、推荐方案总览
**不使用封装请求工具**，只解决“URL集中管理”和“环境自动切换”以及“轻量级全局数据共享”。

| 目标                  | 方案                                                             | 关键 API / 文件                                |
| --------------------- | ---------------------------------------------------------------- | ---------------------------------------------- |
| 统一管理所有 URL      | 创建 `config/api.js`，集中定义 API 路径和基础地址                | `module.exports`                               |
| 自动区分开发/生产环境 | 使用 `wx.getAccountInfoSync()` 获取当前环境，动态拼接 `BASE_URL` | `envVersion` (`develop` / `trial` / `release`) |
| 跨页面共享数据        | 使用 `App.globalData` + 本地缓存（`wx.setStorageSync`）          | `getApp()`                                     |

## 三、具体实现步骤（纯配置 + 全局变量）

### 1. 创建配置文件 `config/api.js`

```javascript
const accountInfo = wx.getAccountInfoSync();
const envVersion = accountInfo.miniProgram.envVersion; // 'develop', 'trial', 'release'

const BASE_URLS = {
  develop: 'https://dev.example.com',   // 开发版（工具/真机预览）
  trial: 'https://test.example.com',    // 体验版（上传后设置体验版）
  release: 'https://api.example.com'    // 正式版（审核上线后）
};

const BASE_URL = BASE_URLS[envVersion];

// 定义所有接口路径（按模块分组可选）
const API = {
  USER_LOGIN: '/user/login',
  USER_INFO: '/user/info',
  ARTICLE_LIST: '/article/list',
  // ...更多
};

module.exports = {
  BASE_URL,
  API
};
```

> **使用方式**：在任意 `.js` 文件中引入  
> `const { BASE_URL, API } = require('../../config/api.js');`  
> 然后拼接完整 URL：`const fullUrl = BASE_URL + API.USER_LOGIN;`

### 2. 在 `app.js` 中定义全局数据（globalData）

```javascript
App({
  globalData: {
    userInfo: null,
    token: '',
    apiBase: null   // 可选：动态从后端获取的地址
  },
  onLaunch() {
    // 读取本地缓存的 token
    const token = wx.getStorageSync('token');
    if (token) this.globalData.token = token;
  },
  setToken(token) {
    this.globalData.token = token;
    wx.setStorageSync('token', token); // 持久化
  }
});
```

### 3. 在页面中访问/修改全局数据

```javascript
// 获取 app 实例
const app = getApp();

Page({
  onLoad() {
    // 读取全局 token
    console.log(app.globalData.token);
    
    // 修改全局 token
    app.setToken('new_token_value');
    
    // 读取配置的 URL
    const { BASE_URL, API } = require('../../config/api.js');
    const loginUrl = BASE_URL + API.USER_LOGIN;
  }
});
```

## 四、关键细节与注意事项

### 环境自动切换原理
- `develop` → 开发者工具、真机预览（未上传）
- `trial` → 上传后设为“体验版”
- `release` → 审核通过后的正式版

> **无需任何手动开关**，小程序运行时自动识别。

### 关于后端动态下发 URL（进阶）
如果需要**不重新发版就能更改 API 地址**，可以：
1. 小程序启动时请求一个**固定后端接口**（比如 `https://fixed.example.com/getConfig`）获取最新的 `BASE_URL`。
2. 将获取到的地址存入 `globalData` 或本地缓存。
3. 后续请求都使用动态地址。

> 注意处理好“首次获取配置失败”的兜底方案（使用本地写死的默认地址）。

### 为什么个人项目不需要 MobX / Westore？
- 学习成本高，配置繁琐。
- `globalData` + `setData` 完全能应付中小型项目的状态共享。
- 如果确实需要响应式更新（比如购物车数量在多个页面同步），可以配合 `wx.setStorageSync` + `onShow` 刷新，或者引入 `EventBus` 发布订阅模式。

## 五、总结一句话
> **创建 `config/api.js` 集中管理 URL，用 `wx.getAccountInfoSync()` 自动切换环境，用 `App.globalData` + 本地缓存共享数据。**  
这三个小技巧就能让个人小程序的配置维护变得像企业项目一样规范，而且几乎零成本。

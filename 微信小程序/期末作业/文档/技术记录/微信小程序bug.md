### 📌 问题精准描述：
**微信开发者工具「代码质量」面板中的静态分析模块存在逻辑缺陷。**

*   **错误现象**：工具在扫描“无使用文件”时，将 `components/navigation-bar/navigation-bar.json` 错误地归类为“无使用的组件”，提示“建议去除”。
*   **矛盾点**：同一时刻，工具的**依赖图**（上方）明确显示 `app.json` -> `home` 页面 -> `navigation-bar` 组件。组件已经被正确引用和配置。
*   **根本原因**：工具在进行“代码质量（静态扫描）”时，对组件 JSON 文件的**识别逻辑**和依赖图的“构建逻辑”是**分离的**。扫描器没有正确解析 `navigation-bar.json` 中的 `"component": true` 以及它在 `home.json` 中的引用关系。即使清缓存和重启，这种静态扫描的错误判定依然存在于其计算规则中。

### 🚨 为什么绝对不要按“建议去除”？
如果你点击去除，删除了这个 `json` 文件，你的 `navigation-bar` 组件会立刻失效，控制台会疯狂报错：“找不到组件定义 `navigation-bar`”。因为微信小程序**必须**通过 `navigation-bar.json` 中的 `"component": true` 来识别这是一个组件。**删掉它，组件就没了灵魂。**

### 🛠️ 最有效的终极对策（无视它+隐藏它）
既然是IDE的Bug，我们不需要通过改代码来适配它，我们只需要**屏蔽掉这个烦人的提示**。

#### 1. 最推荐的懒人方法：隐藏该面板
*   在开发者工具底部，直接找到 `代码质量` 这个标签，点 `X` 关掉它。
*   或者在该面板右上角点击 `...`（更多操作），选择 **“关闭代码质量面板”** 或 **“忽略此规则”**（如果找不到忽略，就直接关掉面板）。  
**只要模拟器和真机上运行正常，`navigation-bar` 能正常显示，那个报错就完全不需要管。**

#### 2. 如果你一定要在「代码质量」里“清清白白”：
这个方法能强制工具跳过对这个文件的扫描。请尝试在项目根目录（和 `app.json` 同级）创建一个名为 `.eslintignore` 的文件（如果已有则直接添加），内容如下：

```text
# 忽略 navigation-bar 组件的 JSON 扫描错误
components/navigation-bar/navigation-bar.json
```

保存后，在开发者工具顶部菜单点击 **工具 -> 构建 npm** 或 **重新扫描代码质量**，这个错大概率就会消失。

# 微信小程序：如何通过云开发配置 SMTP 发送邮件

在微信小程序中，发送邮件的最佳实践是利用**微信云开发（CloudBase）**。由于小程序前端环境无法直接存储密钥且无法使用受限的 Node.js 模块，必须通过后端云函数来处理。

### 一、 核心架构流程

1. **前端**：触发发送请求，传递目标邮箱。
2. **云函数**：接收请求，配置 `nodemailer`，执行发送任务。
3. **SMTP 服务器**：QQ 邮箱服务器接收指令并投递邮件。

### 二、 配置前准备

在开始代码编写前，请务必完成以下工作：

* **开通云开发**：在微信开发者工具中确保项目已开启云开发能力。
* **获取 SMTP 授权码**：
1. 登录电脑版 QQ 邮箱。
2. 点击 **“设置” -> “账号”**。
3. 找到 **POP3/IMAP/SMTP/Exchange/CardDAV/CalDAV 服务**，开启对应服务。
4. 生成并记录 **16位授权码**（这不再是你的 QQ 密码）。



### 三、 项目结构配置

在你的 `cloudfunctions` 目录下创建 `sendEmail` 函数，确保结构如下：

```text
cloudfunctions/
└── sendEmail/
    ├── index.js          // 云函数核心逻辑
    ├── package.json      // 依赖声明文件
    └── node_modules/     // 执行 npm install 后生成的库文件

```

**必须修改 `package.json**`：
确保 `dependencies` 中包含 `nodemailer`，否则云端无法运行。

```json
{
  "dependencies": {
    "wx-server-sdk": "~3.0.4",
    "nodemailer": "^6.9.0"
  }
}

```

### 四、 代码实现

#### 1. 云函数 `index.js`

```javascript
const cloud = require('wx-server-sdk')
const nodemailer = require('nodemailer')

cloud.init({ env: cloud.DYNAMIC_CURRENT_ENV })

exports.main = async (event, context) => {
  const code = Math.floor(1000 + Math.random() * 9000).toString() // 生成4位验证码

  const transporter = nodemailer.createTransport({
    host: 'smtp.qq.com',
    port: 465,
    secure: true, 
    auth: {
      user: '你的QQ邮箱@qq.com',
      pass: '你的SMTP授权码' // 使用生成的授权码
    }
  })

  try {
    await transporter.sendMail({
      from: '"验证服务" <你的QQ邮箱@qq.com>',
      to: event.email,
      subject: '您的验证码',
      text: `您的验证码是：${code}，有效期5分钟。`,
    })
    return { success: true }
  } catch (e) {
    return { success: false, error: e }
  }
}

```

### 五、 部署与注意事项

* **全量部署**：修改 `package.json` 或添加 `nodemailer` 后，必须在开发者工具中对该云函数文件夹点击 **“右键 -> 上传并部署：所有文件”**。
* **安全性建议**：
* **授权码保护**：不要将授权码上传到 GitHub 等公共仓库。
* **频率限制**：在云函数中添加逻辑（例如记录用户最后发送时间），防止被恶意刷邮件导致 QQ 邮箱被封。
* **验证码校验**：建议配合云数据库，将验证码存储在数据库中并设置 `expireAt`（自动过期时间），以确保校验的安全性。

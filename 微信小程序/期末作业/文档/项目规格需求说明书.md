## 1. 引言

### 1.1 编写目的
本文档定义"ACG智能图廊"微信小程序的完整软件需求，作为系统设计、编码实现和测试验收的共同基线。预期读者包括开发人员、测试人员、UI设计师及项目管理者。

### 1.2 产品范围
本产品是一个面向ACG爱好者与图片收藏者的微信小程序，为用户提供主题分类浏览、个人收藏管理、基础账号功能，以及**基于 Dify 大模型平台的 AI 图片鉴赏对话**——用户可上传图片或输入文字，与 AI 进行多模态对话，获取图片分析、风格解读、创作建议等智能服务。系统包含完整的 AI 会话管理（历史记录、多轮对话、反馈评价）。传统图片资源由后台管理提供，AI 对话中的图片由用户实时上传。

### 1.3 定义、缩略语与术语
| 术语          | 说明                                                      |
| ------------- | --------------------------------------------------------- |
| ACG           | Animation（动画）、Comic（漫画）、Game（游戏）的缩写      |
| AI描述        | 通过机器学习模型预先生成的对图片内容的文字描述            |
| AI标签        | 通过模型预先生成的图片分类关键词数组                      |
| 主题          | 图片的一种分类维度，如"ACG"、"PC壁纸"、"风景"、"AI绘图"等 |
| 自然语言搜索  | 用户输入自然语句（如"雨中的樱花"），系统返回相关图片      |
| JWT           | JSON Web Token，无状态身份认证令牌                        |
| Access Token  | 短期访问令牌（15分钟），每次 API 请求在 Header 中携带     |
| Refresh Token | 长期刷新令牌（30天），存储于 Redis，用于无感刷新          |
| Dify          | 开源 LLM 应用开发平台，本项目使用其对话型应用 API         |
| SSE           | Server-Sent Events，服务端推送事件流，用于 AI 流式输出    |
| 多模态对话    | AI 同时处理文本和图片输入，生成文本回复的能力             |
| Agent 思考    | AI 在回答前的推理步骤（thought/observation/tool 等）      |

## 2. 总体描述

### 2.1 产品视角
ACG智能图廊作为独立微信小程序运行，使用邮箱验证码登录。系统与以下外部实体交互：
- **微信客户端**：提供小程序运行容器、震动反馈等开放能力。
- **后端服务（FastAPI + MongoDB + Redis）**：
  - **数据库服务（mongodb_service，端口 21325）**：处理邮箱验证码登录、JWT 双 Token 认证、图片数据查询、浏览/收藏记录存储。
  - **AI 服务（Dify 对话型应用，端口 21326）**：基于 Dify 平台的 AI 对话引擎，支持 SSE 流式输出、多模态图片理解、会话持久化。通过 Nginx `/ai/*` 路由访问，API Key 鉴权。用户标识通过数据库服务 `GET /auth/me` 获取。
- **Nginx 反向代理**：通过 Docker Compose 部署，统一入口（宿主机 61000 端口），按 URL 前缀 `/db/*`、`/ai/*` 分流请求至对应后端服务。
- **Ngrok 内网穿透**：将本地 Nginx 的 61000 端口映射至公网 HTTPS 地址，供微信小程序调用。
- **图片存储服务**：用于存储原图与缩略图（如云存储或CDN）。

上下文图（文字描述）：
```
[用户] <--> (微信小程序) <--> [Ngrok HTTPS 公网] <--> [Nginx 反向代理 :61000]
                                                            │
                                    ┌───────────────────────┤
                                    │ /db/*                  │ /ai/*
                                    ▼                        ▼
                            [数据库服务 :21325]        [AI 服务 :21326]
                            FastAPI                    FastAPI
                            ├── MongoDB（用户+图片）    ├── 大模型 API
                            │   ├── users（含浏览/收藏）  │
                            │   └── images                │
                            └── Redis                     │
                                ├── Refresh Token          │
                                ├── 邮箱验证码              │
                                └── 发送冷却期              │
```

### 2.2 产品功能概要
- **邮箱验证码登录**：用户输入邮箱 → 获取6位验证码 → 验证登录，新用户自动注册（✅ 已实现）
- **JWT 双 Token 认证**：Access Token（15min）+ Refresh Token（30day），Token Rotation 防重放（✅ 已实现）
- **浏览记录**：自动记录浏览历史，最多1000条，分页查询（✅ 后端已实现）
- **收藏管理**：收藏/取消/列表/清空，嵌入用户文档（✅ 后端已实现）
- **主题导航**：首页展示图片分类入口（← 这是微信小程序的首页，需要补充界面功能）
- **图片网格浏览**：以缩略图网格分页展示图片（✅ 已实现）
- **图片详情弹窗**：点击缩略图弹出大图、AI描述、标签和收藏按钮（✅ 已实现）
- **智能搜索**：用户输入自然语言文本，返回匹配的图片结果（（✅ 已实现）
- **AI 图片鉴赏对话**：用户上传图片或输入文字，与 Dify 大模型进行多模态对话，获取图片分析、风格解读等智能回复（✅ 已实现）
- **AI SSE 流式输出**：对话采用 SSE 流式协议，AI 回复逐字输出，支持随时停止生成（✅ 已实现）
- **AI 会话管理**：支持多轮对话、历史会话列表、会话切换/删除，<think> 思考过程折叠展示（✅ 已实现）
- **AI 消息反馈**：支持对 AI 回复点赞/点踩，反馈数据回传 Dify（✅ 已实现）
- **未登录拦截**：需登录的操作自动跳转登录页（✅ 已实现）

### 2.3 用户特征与角色
| 角色               | 特征                                                       | 相关需求            |
| ------------------ | ---------------------------------------------------------- | ------------------- |
| 游客（未登录）     | 仅可浏览首页主题列表和图片列表，不能使用搜索、收藏、设置页 | 受限浏览            |
| 注册用户（已登录） | 可使用所有功能，拥有个人收藏和账号管理能力                 | FR-01, FR-05～FR-12 |

### 2.4 运行环境
- **微信客户端**：基础库版本 ≥ 2.10.0，支持 iOS 12+ 和 Android 8.0+。
- **网络环境**：4G/Wi-Fi。
- **后端环境**：Python 3.14+，FastAPI + Uvicorn，MongoDB + Redis。
- **反向代理**：Nginx（Docker 容器化部署），对外暴露 61000 端口。
- **内网穿透**：Ngrok，将本地 61000 端口映射至公网 HTTPS 地址。

### 2.5 设计与实现约束
- 必须使用微信小程序原生框架开发，不能使用 WebView 套壳。
- 必须遵循微信小程序包大小限制（主包 ≤ 2MB，总包 ≤ 20MB）。
- UI 风格须为粉白柔色系、卡片圆角、毛玻璃效果，符合 ACG 清新视觉。
- 用户认证使用邮箱验证码登录，不使用微信授权登录。
- 图片数据来源为后台管理员通过导入脚本批量入库。AI 对话中用户可上传图片供大模型分析（上传至 Dify，不存入 MongoDB 图片库）。
- 后端使用 Nginx 反向代理 + Ngrok 内网穿透提供公网 HTTPS 访问。
- 后端不使用 ODM，使用 pymongo 原生异步 + pydantic BaseModel 做数据校验。
- Redis 使用 RESP2 协议（`protocol=2`），规避 redis-py 6.x 的 RESP3 认证时序问题。

### 2.6 假设与依赖
- 假设用户拥有有效的邮箱地址用于接收验证码。
- 依赖后端 AI 服务已预先生成所有图片的描述与标签，并随图片数据一起存入数据库。
- 自然语言搜索依赖后端搜索引擎或数据库全文索引能力，返回结果时间 ≤ 3 秒。
- 图片资源已裁剪缩略图，缩略图尺寸统一为宽度 400px，保持比例。
- 依赖 Redis 服务可用（用于 Refresh Token 存储、验证码存储、冷却期控制）。
- 依赖 QQ 邮箱 SMTP 服务用于发送验证码。
- 依赖 Docker Desktop 运行 Nginx 容器。
- 依赖 Ngrok 客户端提供公网 HTTPS 隧道（微信小程序要求合法域名）。

## 3. 具体需求

### 3.1 功能需求

本章节所有功能需求均以 **"系统必须"** 的句式描述可测试的行为，每项需求包含：编号、名称、输入、处理逻辑、输出、前置条件、后置条件及验收标准。

#### 3.1.1 登录与注册

**FR-01 邮箱验证码登录**
- **输入**：用户在登录页输入邮箱地址，点击"发送验证码"；收到验证码后输入6位数字，点击"登录"。
- **处理**：
  1. 发送验证码（`POST /auth/send-code`）：后端检查60秒冷却期 → 生成6位随机数字 → 存入 Redis（5分钟有效）→ 通过 QQ 邮箱 SMTP 发送 → 设置冷却期标记
  2. 登录（`POST /auth/login`）：后端校验验证码（一次性消费，验证后立即从 Redis 删除）→ 查找用户 → 不存在则自动创建新用户（验证码即注册）→ 签发 Access Token（15分钟）+ Refresh Token（30天）→ Refresh Token 的 jti 存入 Redis
- **输出**：返回 `access_token`、`refresh_token`、`token_type`（"bearer"）、`expires_in`（900秒）。客户端存储 token，后续请求在 Header 中携带 `Authorization: Bearer <access_token>`。
- **前置条件**：用户未登录，邮箱地址有效。
- **后置条件**：客户端持有效 token 对，Refresh Token 在 Redis 中可追踪和撤销。
- **验收标准**：
  1. 发送验证码后，60秒内再次请求返回 429（HTTP 状态码），提示冷却剩余秒数。
  2. 验证码 5 分钟内有效，过期后返回 401。
  3. 验证码正确 → 登录成功，2 秒内完成。
  4. 新用户首次登录自动创建账号，再次登录数据完整保留。
  5. Access Token 过期（15分钟）后，API 返回 401 + 响应头 `X-Token-Error: expired`。
  6. 验证码为一次性使用，用后即删。

#### 3.1.2 首页主题导航

**FR-02 展示主题列表**（← 这是微信小程序的首页，需要补充界面功能）
- **输入**：用户进入首页，或从登录页跳转至此。
- **处理**：系统请求后端获取主题配置列表（若网络异常则使用本地缓存的主题列表）。每个主题具有名称和图标。
- **输出**：页面渲染包含图标的主题卡片，点击任一卡片跳转到图片鉴赏页并携带 `topic` 参数。
- **前置条件**：用户已登录或处于游客模式。
- **验收标准**：
  1. 首页稳定显示主题卡片，布局无错位。
  2. 无网络时展示本地兜底主题列表。

#### 3.1.3 图片鉴赏页（图片网格列表与详情）

**FR-03 按主题加载图片列表**（← 需要补充界面功能与后端 /images 接口）
- **输入**：页面接收参数 `topic`。首次加载分页参数。
- **处理**：后端根据主题查询图片集合，返回缩略图URL、AI描述、图片ID等字段。
- **输出**：页面以 2 列网格展示图片缩略图，滚动到底部自动加载下一页。
- **验收标准**：首屏 10 张缩略图在 2 秒内显示；上拉到底自动追加；网络错误时显示重试按钮。

**FR-04 图片详情弹窗**（← 需要补充界面功能）
- **输入**：用户点击网格中的某张缩略图。
- **处理**：获取该图片完整信息（原图URL、AI描述、AI标签、收藏状态）。弹出模态窗口展示。
- **输出**：弹窗渲染，点击遮罩或关闭按钮可关闭。

**FR-05 收藏/取消收藏功能**（← 后端 API 已就绪，前端需要补充界面功能）
- **输入**：用户在图片详情弹窗中点击"收藏"按钮。
- **处理**：调用 `POST /favorite/`（添加）或 `DELETE /favorite/{image_id}`（取消）。后端使用 `$pull` + `$push` 实现幂等添加，嵌入在 User 文档的 `favorite_records` 数组中，无上限，按 image_id 降序排列。
- **输出**：按钮状态切换 + toast 提示 + 短震动反馈。
- **前置条件**：用户已登录。
- **验收标准**：
  1. 同一用户对同一图片不会出现重复收藏。
  2. 关闭弹窗再重新打开，收藏状态保持。

**FR-06 AI 图片鉴赏对话**
- **输入**：用户在搜索/探索页输入文字，或点击上传按钮选择图片（最多3张），点击发送。
- **处理**：
  1. 若有图片 → 调用 Dify 文件上传 API（`POST /ai/v1/files/upload`，multipart/form-data），拿到 `upload_file_id`
  2. 构造请求体：`{ query, user, conversation_id, files: [{ type, transfer_method, upload_file_id }], response_mode: "streaming" }`
  3. 调用 Dify 对话 API（`POST /ai/v1/chat-messages`），`enableChunked: true`
  4. 解析 SSE 流：`message` 事件 → `answer` 字段逐 token 追加显示；`message_end` → 标记完成；`error` → 显示错误
  5. 首次对话返回 `conversation_id`，保存用于后续多轮对话
- **输出**：AI 回复以流式打字机效果逐字显示，末尾闪烁光标 ▊。用户可点击停止按钮中断生成。
- **前置条件**：用户已登录，`userId` 已从数据库服务获取。
- **后置条件**：会话记录保存至 Dify，可在历史会话列表中查看。
- **验收标准**：
  1. 纯文本消息发送后，AI 回复在 1 秒内开始流式输出（首 token 延迟）。
  2. 图片上传 + 发送在 5 秒内完成（含上传耗时）。
  3. 流式输出过程中点击停止，生成立即中断并保留已输出内容。
  4. 网络异常时显示错误提示，支持点击"重试"重新发送。

**FR-13 AI 会话历史管理**
- **输入**：用户点击顶部导航栏左侧菜单图标，打开侧边栏；或点击某一历史会话。
- **处理**：
  1. 侧边栏调用 `GET /ai/v1/conversations?user={userId}` 获取会话列表（按更新时间倒序）
  2. 点击某会话 → 调用 `GET /ai/v1/messages?user={userId}&conversation_id={id}` 获取历史消息（倒序返回，前端反转为正序）
  3. 支持左滑删除：调用 `DELETE /ai/v1/conversations/{id}`，Body `{ user }`
  4. 点击"新对话"按钮清空当前会话上下文，下次发送消息创建新会话
- **输出**：侧边栏展示会话名称和更新时间；点击加载对应历史消息到聊天区；删除后从列表消失。
- **验收标准**：
  1. 会话列表在 1 秒内加载完成（Wi-Fi 环境）。
  2. 历史消息加载后显示顺序为旧→新，用户和 AI 消息交替排列。
  3. 删除当前会话后自动新建空白会话。
  4. 空会话列表时显示"暂无历史对话"。

**FR-14 AI 思考过程展示**
- **输入**：AI 回复中包含 `<think>...</think>` 标签包裹的思考内容，或 Dify Agent 模式推送的 `agent_thought` SSE 事件。
- **处理**：
  1. SSE 流接收时实时剥离 `<think>` 标签内容，存入 `thought` 字段，正文存入 `content`
  2. Dify `agent_thought` 事件 → 拼接 `thought/observation/tool` 到思考字段
  3. 默认折叠：显示灰色胶囊按钮 "▸ 💭 思考过程"，点击展开显示灰色代码风格文本块
  4. 思考内容区 `max-height: 400rpx`，超出可滚动
- **验收标准**：
  1. 包含 `<think>` 标签的回复，思考内容不显示在正文中。
  2. 默认折叠，用户点击后展开/收起，状态切换流畅。
  3. 展开后的思考内容可选中复制。

**FR-15 AI 消息反馈（点赞/点踩）**
- **输入**：用户在 AI 回复下方点击 👍 或 👎。
- **处理**：
  1. 调用 `POST /ai/v1/messages/{message_id}/feedbacks`，Body `{ rating: "like"|"dislike", user, content }`
  2. `message_id` 优先使用 SSE 流返回的 Dify 真实 ID（`difyMessageId`），历史消息使用加载时的 `item.id`
  3. 本地更新消息状态（高亮已选按钮）
- **验收标准**：
  1. 点赞/点踩后按钮高亮，再次点击同一按钮取消反馈。
  2. 反馈数据正确回传至 Dify 后台可查。

**FR-16 AI 回复复制**
- **输入**：用户点击 AI 回复下方的 📋 按钮。
- **处理**：调用 `wx.setClipboardData` 复制消息原文，toast 提示"已复制"。
- **验收标准**：复制后内容与显示完全一致（含 Markdown 原始文本）。

**FR-17 AI 回复重新生成**
- **输入**：用户点击 AI 回复下方的 🔄 按钮。
- **处理**：找到上一条用户消息（内容+附件），删除当前 AI 回复，以相同参数重新调用 Dify 对话 API。
- **验收标准**：
  1. 重新生成时显示思考中动画，完成后新回复替换旧回复。
  2. 生成期间不可再次点击重新生成。

#### 3.1.4 个人设置与记录管理

**FR-07 个人档案展示**（← 需要补充界面功能）
- **输入**：用户进入设置页。
- **处理**：从后端 `GET /auth/me` 获取当前用户信息（id、email、username、创建时间、更新时间）。
- **输出**：页面展示用户基本信息。
- **前置条件**：用户已登录。
- **验收标准**：用户信息正确显示。

**FR-08 我的浏览记录**（← 后端 API 已就绪，前端需要补充界面功能）
- **输入**：用户在设置页点击"浏览记录"入口。
- **处理**：调用 `GET /browse/?limit=20&skip=0` 分页获取。后端使用 MongoDB `$slice: [skip, limit]` 在服务端分页，返回按 image_id 降序（新→旧）排列的记录。最多保留 1000 条。
- **输出**：展示图片列表，支持分页加载。
- **验收标准**：
  1. 空记录时显示"暂无浏览记录"。
  2. 每次请求最多 100 条（硬上限），靠 skip 翻页。
  3. 最新浏览的图片排在最前面。

**FR-09 我的收藏列表**（← 后端 API 已就绪，前端需要补充界面功能）
- **输入**：用户在设置页点击"我的收藏"入口。
- **处理**：调用 `GET /favorite/?limit=20&skip=0` 分页获取。后端使用 MongoDB `$slice: [skip, limit]` 分页，按 image_id 降序排列。无上限。
- **输出**：展示图片网格，可取消收藏。
- **验收标准**：
  1. 空收藏时显示"还没有收藏任何图片"。
  2. 取消收藏后图片即时消失。

**FR-10 退出登录**
- **输入**：用户在设置页点击"退出登录"。
- **处理**：弹出二次确认弹窗，确认后调用 `POST /auth/logout`（可携带 refresh_token 实现单设备登出，不携带则全设备登出），清除客户端 token 和本地缓存数据，跳转至登录页。不删除服务端用户数据。
- **前置条件**：用户已登录。
- **验收标准**：
  1. 退出后无法通过"我的"Tab 直接进入（自动跳转登录页）。
  2. 重新登录后，之前的收藏和浏览数据仍保留。

#### 3.1.5 全局与拦截

**FR-11 未登录拦截跳转**（← 前端拦截器需要补充实现）
- **输入**：用户未登录时点击需要登录的页面或功能。
- **处理**：系统检测本地无有效 token，自动跳转登录页。
- **验收标准**：未登录时访问需认证页面，瞬间跳转登录页，无白屏。

**FR-12 Token 无感刷新**（← 前端拦截器需要补充实现）
- **输入**：API 请求返回 401 + `X-Token-Error: expired`。
- **处理**：前端拦截器自动调用 `POST /auth/refresh`，用 Refresh Token 换取新 Token 对（Token Rotation），重试原请求。
- **验收标准**：
  1. 用户无感知，不会因 Access Token 过期而中断使用。
  2. 若 Refresh Token 也过期或检测到重放攻击（401 + `X-Token-Error: reused`），跳转登录页要求重新验证。

### 3.3 数据需求（逻辑数据模型）

> 物理实现采用 MongoDB 嵌入式文档模式，浏览和收藏记录嵌入在用户文档中。

#### 3.3.1 实体与属性定义

**用户（User）—— MongoDB `users` 集合**
| 属性               | 类型                  | 约束         | 说明                             |
| ------------------ | --------------------- | ------------ | -------------------------------- |
| `_id`              | ObjectId              | PK，自动生成 | 代理主键（应用层转为 `id: str`） |
| `email`            | String                | UNIQUE       | 登录凭证，邮箱格式               |
| `username`         | String                | —            | 用户昵称（默认取邮箱）           |
| `browse_records`   | Array\<BrowseItem\>   | 最多 1000 条 | 嵌入式浏览记录，image_id 降序    |
| `favorite_records` | Array\<FavoriteItem\> | 无上限       | 嵌入式收藏记录，image_id 降序    |
| `created_at`       | DateTime              | 自动生成     | 注册时间                         |
| `updated_at`       | DateTime              | 自动生成     | 最近更新时间                     |

**BrowseItem 子文档（嵌入在 User.browse_records）**
| 属性          | 类型     | 说明                                      |
| ------------- | -------- | ----------------------------------------- |
| `image_id`    | String   | 图片唯一标识（时间戳，如 20250610120000） |
| `image_url`   | String   | 图片 URL（冗余，避免 JOIN images 集合）   |
| `browse_time` | DateTime | 浏览时间                                  |

**FavoriteItem 子文档（嵌入在 User.favorite_records）**
| 属性            | 类型     | 说明                        |
| --------------- | -------- | --------------------------- |
| `image_id`      | String   | 图片唯一标识（时间戳）      |
| `image_url`     | String   | 图片 URL（冗余，避免 JOIN） |
| `favorite_time` | DateTime | 收藏时间                    |

**图片（Image）—— MongoDB `images` 集合**
| 属性         | 类型     | 约束     | 说明                                  |
| ------------ | -------- | -------- | ------------------------------------- |
| `_id`        | String   | PK       | 时间戳作为主键（如 `20250610120000`） |
| `url`        | String   | UNIQUE   | 图片资源地址（OSS/CDN）               |
| `type`       | String   | —        | 图片格式（jpg, png, webp, gif）       |
| `created_at` | DateTime | 自动生成 | 入库时间                              |

**业务规则**：
- 浏览记录通过 `$push` + `$each` + `$sort: {image_id: DESCENDING}` + `$slice: -1000` 维护，自动裁剪旧数据。
- 收藏记录添加时先 `$pull`（按 image_id 去重）再 `$push` + `$sort`（降序），保证幂等。
- 图片 URL 在浏览/收藏子文档中冗余存储，查询历史列表时无需 JOIN images 集合。
- 分页查询使用 MongoDB `$slice: [skip, limit]`，服务端截断，不会拉取全量数据。
- JWT Token 载荷中仅含 `sub`（用户 ID），不含 email 等业务字段。

#### 3.3.2 数据字典补充说明
- `image_id` 为时间戳字符串（如 `20250610120000`），利于排序。
- 所有 URL 字段在生产环境必须通过 HTTPS 访问。

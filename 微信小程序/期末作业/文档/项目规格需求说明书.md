## 1. 引言

### 1.1 编写目的
本文档定义“ACG智能图廊”微信小程序的完整软件需求，作为系统设计、编码实现和测试验收的共同基线。预期读者包括开发人员、测试人员、UI设计师及项目管理者。

### 1.2 产品范围
本产品是一个面向ACG爱好者与图片收藏者的微信小程序，为用户提供主题分类浏览、自然语言搜索图片、图片详情查看（含AI描述与标签）、个人收藏管理及基础账号功能。系统不包含图片上传或用户生成内容功能，所有图片资源由后台管理提供。

### 1.3 定义、缩略语与术语
| 术语         | 说明                                                      |
| ------------ | --------------------------------------------------------- |
| ACG          | Animation（动画）、Comic（漫画）、Game（游戏）的缩写      |
| AI描述       | 通过机器学习模型预先生成的对图片内容的文字描述            |
| AI标签       | 通过模型预先生成的图片分类关键词数组                      |
| 主题         | 图片的一种分类维度，如“ACG”、“PC壁纸”、“风景”、“AI绘图”等 |
| 自然语言搜索 | 用户输入自然语句（如“雨中的樱花”），系统返回相关图片      |

## 2. 总体描述

### 2.1 产品视角
ACG智能图廊作为独立微信小程序运行，不集成外部第三方登录（仅使用微信授权登录）。系统与以下外部实体交互：
- **微信客户端**：提供登录授权、相册/相机调用（若未来需要）、震动反馈等开放能力。
- **后端服务（云函数或自建API）**：处理图片数据查询、自然语言搜索、收藏关系存储、用户信息管理。
- **图片存储服务**：用于存储原图与缩略图（如云存储或CDN）。
- **AI标签/描述服务**：在图片入库时离线生成描述与标签，不在客户端实时调用。

上下文图（文字描述）：
```
[用户] <--> (微信小程序) <--> [后端API服务] <--> [数据库]
                                    <--> [AI服务(离线批处理)]
```

### 2.2 产品功能概要
- **主题导航**：首页展示图片分类入口，用户选择主题后进入对应图片列表。
- **图片网格浏览**：以缩略图网格分页展示图片，支持上拉加载更多。
- **图片详情弹窗**：点击缩略图弹出大图、AI描述、标签和收藏按钮。
- **智能搜索**：用户输入自然语言文本，返回匹配的图片结果。
- **个人收藏**：用户可收藏图片，在“我的收藏”中统一查看。
- **账号管理**：微信一键登录/注册，个人信息展示，退出登录。
- **未登录拦截**：需登录的操作自动跳转登录页。

### 2.3 用户特征与角色
| 角色               | 特征                                                       | 相关需求            |
| ------------------ | ---------------------------------------------------------- | ------------------- |
| 游客（未登录）     | 仅可浏览首页主题列表和图片列表，不能使用搜索、收藏、设置页 | 受限浏览            |
| 注册用户（已登录） | 可使用所有功能，拥有个人收藏和账号管理能力                 | FR-01, FR-05～FR-12 |

用户群体为ACG爱好者，熟悉移动端操作，对响应速度敏感。

### 2.4 运行环境
- **微信客户端**：基础库版本 ≥ 2.10.0，支持 iOS 12+ 和 Android 8.0+。
- **网络环境**：4G/Wi-Fi。
- **后端环境**：Node.js 云函数 或 传统 RESTful 服务器，数据库采用 NoSQL 文档数据库（如微信云开发数据库或 MongoDB）。

### 2.5 设计与实现约束
- 必须使用微信小程序原生框架开发，不能使用 WebView 套壳。
- 必须遵循微信小程序包大小限制（主包 ≤ 2MB，总包 ≤ 20MB）。
- UI 风格须为粉白柔色系、卡片圆角、毛玻璃效果，符合 ACG 清新视觉。
- 用户认证仅使用微信登录，不允许其他第三方登录。
- 图片数据来源为后台管理员通过导入脚本批量入库，小程序端不提供上传入口。

### 2.6 假设与依赖
- 假设用户已安装微信并授权获取昵称、头像等基本信息。
- 依赖后端 AI 服务已预先生成所有图片的描述与标签，并随图片数据一起存入数据库。
- 自然语言搜索依赖后端搜索引擎或数据库全文索引能力，返回结果时间 ≤ 3 秒。
- 图片资源已裁剪缩略图，缩略图尺寸统一为宽度 400px，保持比例。

## 3. 具体需求

### 3.1 功能需求

本章节所有功能需求均以 **“系统必须”** 的句式描述可测试的行为，每项需求包含：编号、名称、输入、处理逻辑、输出、前置条件、后置条件及验收标准。

#### 3.1.1 登录与注册

**FR-01 微信授权登录**
- **输入**：用户点击“微信登录”按钮，触发 `wx.getUserProfile`（或静默授权）。
- **处理**：系统获取用户微信昵称、头像、openid，将 openid 发送后端。后端若不存在该用户则自动创建用户记录，并返回登录态 token。若用户已存在则更新昵称和头像。
- **输出**：token 存储在客户端本地 Storage。页面跳转至首页。
- **前置条件**：用户未登录。
- **后置条件**：客户端持有效 token，所有后续 API 请求在 Header 中携带 `Authorization: Bearer <token>`。
- **验收标准**：
  1. 点击按钮后弹出微信授权弹窗。
  2. 授权成功后 2 秒内完成登录并跳转首页。
  3. 若用户拒绝授权，停留在登录页并提示“需要授权才能使用”。
  4. token 有效期 7 天，过期后自动退出登录并回到登录页。

#### 3.1.2 首页主题导航

**FR-02 展示主题列表**
- **输入**：用户进入首页（`pages/home/home`），或从登录页跳转至此。
- **处理**：系统请求后端获取主题配置列表（若网络异常则使用本地缓存的主题列表，至少包含 ACG、PC壁纸、风景、AI绘图 四个主题）。每个主题具有名称和图标。
- **输出**：页面渲染包含图标的主题卡片，卡片数量 ≥ 4，点击任一卡片跳转到图片鉴赏页并携带 `topic` 参数。
- **前置条件**：用户已登录或处于游客模式（游客可看主题但收藏功能受限）。
- **后置条件**：导航到 `/pages/gallery/gallery?topic=<主题标识>`。
- **验收标准**：
  1. 首页稳定显示至少 4 个主题卡片，布局无错位。
  2. 点击“ACG”卡片后成功跳转且页面标题显示为“ACG”。
  3. 无网络时展示本地兜底主题列表。

#### 3.1.3 图片鉴赏页（图片网格列表与详情）

**FR-03 按主题加载图片列表**
- **输入**：页面 `onLoad` 接收参数 `topic` 或 `query`（当来自搜索页时）。首次加载请求参数：`{ topic: "acg", page: 1, pageSize: 10 }`。
- **处理**：后端根据主题或搜索关键词查询图片集合，返回缩略图URL、AI描述（截断为两行预览）、图片ID等字段。分页游标使用页码或最后一条记录的ID。
- **输出**：页面以 2 列网格展示图片缩略图，每张图片下方显示不超过两行的AI描述文本。滚动到底部时自动加载下一页，加载时底部显示 loading 指示器。
- **前置条件**：用户已进入该页面。
- **后置条件**：图片列表数据填充到视图，若数据为空则显示“暂无图片”占位图。
- **验收标准**：
  1. 4G 网络下首屏 10 张缩略图在 2 秒内显示。
  2. 上拉到底自动追加下一页数据，无重复图片。
  3. 网络错误时显示重试按钮，点击后重新请求当前页。
  4. AI描述超过两行时，以省略号截断。

**FR-04 图片详情弹窗**
- **输入**：用户点击网格中的某张缩略图。
- **处理**：系统获取该图片完整的原图URL、AI描述、AI标签数组、收藏状态（当前用户是否已收藏）。弹出模态窗口，内部展示原图（支持双指缩放）、描述全文、标签列表、收藏/取消收藏按钮。
- **输出**：弹窗渲染，点击遮罩或关闭按钮可关闭弹窗。
- **前置条件**：图片缩略图已加载且可点击。
- **后置条件**：弹窗关闭后返回网格列表，列表位置不变。
- **验收标准**：
  1. 弹窗动画流畅，300ms 内出现。
  2. 标签显示为圆角标签样式，至少 1 个标签。
  3. 若用户已登录，收藏按钮状态与实际收藏关系一致（已收藏高亮）。

**FR-05 收藏/取消收藏功能**
- **输入**：用户在图片详情弹窗中点击“收藏”按钮（或后续可能的网格快捷按钮）。
- **处理**：系统发送收藏 toggle 请求至后端 `POST /api/favorites/toggle`，携带图片ID。后端根据当前用户与图片ID判断：若无记录则新增一条收藏文档；若已有则删除该收藏记录。
- **输出**：
  - 按钮文案/样式改变为“已收藏”或“收藏”。
  - 触发手机短震动（`wx.vibrateShort`）及 toast 提示“已收藏”/“已取消收藏”。
  - 如果是从“我的收藏”页取消收藏，该图片卡片应从当前列表动画移除。
- **前置条件**：用户已登录，否则点击时跳转登录页（参见 FR-13）。
- **后置条件**：数据库中的收藏关系与按钮状态同步。
- **验收标准**：
  1. 点击收藏后，震动和 toast 均触发。
  2. 关闭弹窗再重新打开，收藏状态保持。
  3. 同一用户对同一图片不会出现重复收藏记录。

**FR-06 智能搜索功能**
- **输入**：用户在搜索页输入自然语言文本（如“夕阳下的猫”），点击搜索按钮或回车。
- **处理**：系统将文本发送至后端 `/api/search?query=夕阳下的猫`。后端采用全文索引或语义匹配算法检索图片数据库中 AI描述 和 AI标签 字段，按相关性排序返回图片列表。
- **输出**：搜索页下方的图片网格组件更新为搜索结果。若无结果，显示“未找到相关图片”。
- **前置条件**：用户已登录（搜索页在 TabBar，需要登录才可查看，见 FR-13）。
- **后置条件**：搜索历史（可选）可本地缓存最近 5 条，但此版本不作强制要求。
- **验收标准**：
  1. 输入“樱花”后搜索，返回的图片结果中包含标签或描述为“樱花”的图片。
  2. 搜索接口响应时间 ≤ 3 秒（4G）。
  3. 空结果时给出明确提示。

#### 3.1.4 个人设置与收藏管理

**FR-07 个人档案展示**
- **输入**：用户点击 TabBar“我的”进入设置页。
- **处理**：页面从后端获取当前用户信息：头像、昵称、UID、个性签名、等级。若无个性签名和等级，显示默认占位文字。
- **输出**：页面顶部展示可点击更换的头像（调用微信头像选择接口），昵称和签名可点击编辑，UID 不可修改。编辑保存后实时更新后端。
- **前置条件**：用户已登录。
- **验收标准**：
  1. 头像、昵称、UID 正确显示。
  2. 点击昵称弹出输入框，保存后主页昵称同步刷新。

**FR-08 我的收藏列表**
- **输入**：用户在设置页点击“我的收藏”入口。
- **处理**：跳转到一个与图片鉴赏页网格布局复用的页面（或内嵌组件），后端接口返回当前用户收藏的所有图片数据。
- **输出**：展示图片网格，每张图片保留详情弹窗功能，但收藏按钮初始态为“已收藏”。点击可取消收藏，图片从列表移除。
- **前置条件**：用户已登录，拥有至少 0 条收藏。
- **验收标准**：
  1. 空收藏时显示“你还没有收藏任何图片”。
  2. 取消收藏后图片即时消失，无残留。

**FR-09 退出登录**
- **输入**：用户在设置页点击“退出登录”。
- **处理**：弹出二次确认弹窗，确认后清除客户端 token 和本地缓存数据，跳转至登录页。不删除服务端用户数据。
- **前置条件**：用户已登录。
- **验收标准**：
  1. 退出后无法通过“我的”Tab 直接进入（自动跳转登录页）。
  2. 重新登录后，之前的收藏数据仍保留。

#### 3.1.5 全局与拦截

**FR-10 未登录拦截跳转**
- **输入**：用户未登录时点击需要登录的页面或功能（搜索 Tab、设置 Tab、收藏按钮）。
- **处理**：系统检测本地无 token，自动 `wx.redirectTo` 或 `wx.navigateTo` 到登录页，并在登录页 URL 参数中携带来源路径。
- **输出**：登录页展示，登录成功后跳转回原始目标页面。
- **前置条件**：用户未登录。
- **验收标准**：
  1. 未登录点击“搜索”Tab，瞬间跳转登录页，无白屏。
  2. 登录后自动回到搜索页。

### 3.2 外部接口需求

#### 3.2.1 用户接口（UI）
- **页面路由**：见文档开头表格，共 5 个注册页面，TabBar 包含首页、搜索、我的三个入口，登录页为独立非 Tab 页。
- **布局规范**：
  - 所有页面背景色 `#FFF5F7`（浅粉白），卡片背景白色带 16rpx 圆角，阴影 `0 2rpx 12rpx rgba(0,0,0,0.08)`。
  - 按钮点击态透明度降低至 0.85，并伴有微缩放动画（scale 0.98）。
  - 页面切换使用 200ms 淡入淡出动画。
- **反馈机制**：
  - 网络请求期间必须显示加载指示器（导航条加载条或骨架屏）。
  - 错误提示使用 toast 或模态弹窗，停留时间 ≥ 2 秒。

#### 3.2.2 硬件接口
- **震动马达**：调用 `wx.vibrateShort({ type: 'light' })`，仅在收藏/取消收藏操作时触发。
- **相机/相册**：当前版本不使用，但预留扩展能力（未来可能用于以图搜图）。

#### 3.2.3 软件接口
- **微信登录 API**：`wx.login` 获取 code，后端换取 openid 和 session_key；结合 `wx.getUserProfile` 获取用户信息（遵守微信最新隐私政策）。
- **后端 API 通用约定**：
  - Base URL：`https://api.acg-gallery.example.com/v1`
  - 认证方式：Header `Authorization: Bearer <token>`
  - 请求格式：`application/json`（上传文件除外）
  - 响应格式：
    ```json
    {
      "code": 0,
      "message": "success",
      "data": { ... }
    }
    ```
  - 分页参数：`page`, `pageSize`，响应中包含 `total`、`hasMore` 字段。
- **具体端点**（逻辑需求，非物理实现）：
  1. **获取主题列表**：`GET /topics` 返回 `[{ id, name, icon }]`
  2. **图片列表**：`GET /images?topic=acg&page=1&pageSize=10` 返回图片对象数组
  3. **搜索图片**：`GET /images/search?query=夕阳` 返回图片数组
  4. **收藏/取消**：`POST /favorites/toggle` body: `{ imageId: "xx" }` 返回 `{ isFavorited: true/false }`
  5. **获取收藏列表**：`GET /favorites?page=1` 返回图片数组
  6. **获取/更新用户信息**：`GET/PUT /user/profile`
  7. **退出登录**：`POST /auth/logout`
- **数据库接口**：使用 NoSQL 集合操作，通过云函数 SDK 或 REST API 间接访问，小程序端不直接操作数据库。

#### 3.2.4 通信接口
- 网络协议：HTTPS，TLS 1.2+
- 超时设置：API 请求超时 10 秒，图片下载超时 15 秒。
- 错误码约定：401 未登录/token过期；404 资源不存在；500 服务器内部错误。

### 3.3 数据需求（逻辑数据模型）

> 本节描述系统需要管理的数据实体及其逻辑结构，不涉及具体数据库索引设计或物理存储细节。

#### 3.3.1 实体与属性定义

**用户（User）**
| 属性      | 类型        | 约束              | 说明         |
| --------- | ----------- | ----------------- | ------------ |
| userId    | String(32)  | 唯一，系统生成    | 用户唯一标识 |
| openid    | String(64)  | 唯一，不可修改    | 微信身份标识 |
| nickname  | String(20)  | 非空，长度 1-20   | 用户昵称     |
| avatarUrl | String(200) | 合法 URL          | 头像图片地址 |
| signature | String(50)  | 可空              | 个性签名     |
| level     | Integer     | 默认 1，范围 1-99 | 用户等级     |
| createdAt | DateTime    | 自动生成          | 注册时间     |

**图片（Image）**
| 属性          | 类型            | 约束                                   | 说明          |
| ------------- | --------------- | -------------------------------------- | ------------- |
| imageId       | String(32)      | 唯一，系统生成                         | 图片主键      |
| title         | String(30)      | 非空                                   | 图片简短标题  |
| originalUrl   | String(200)     | 合法 URL                               | 原图地址      |
| thumbnailUrl  | String(200)     | 合法 URL                               | 缩略图地址    |
| topic         | String(20)      | 枚举：acg,pc_wallpaper,scenery,ai_art… | 主题分类      |
| aiDescription | String(200)     | 非空                                   | AI 生成描述   |
| aiTags        | Array of String | 数组长度 1-10，元素最大 10 字符        | 标签列表      |
| source        | String(20)      | 非空                                   | 来源 API 名称 |
| createdAt     | DateTime        | 入库时间                               |

**收藏（Favorite）**
| 属性       | 类型       | 约束                     | 说明        |
| ---------- | ---------- | ------------------------ | ----------- |
| favoriteId | String(32) | 唯一，系统生成           | 收藏记录 ID |
| userId     | String(32) | 外键，引用 User.userId   | 收藏用户    |
| imageId    | String(32) | 外键，引用 Image.imageId | 被收藏图片  |
| createdAt  | DateTime   | 自动生成                 | 收藏时间    |

**业务规则**：
- 同一用户对同一图片的收藏记录必须唯一（通过应用层或数据库复合唯一约束实现）。
- 删除图片时，关联的收藏记录应级联删除。
- 用户注销时，其收藏记录物理删除。

#### 3.3.2 数据字典补充说明
- `topic` 枚举值从主题配置接口动态获取，初始值包含 `acg`, `pc_wallpaper`, `scenery`, `ai_art`。
- `aiTags` 数组元素为纯文本，不包含特殊符号，前台展示时用“#”连接。
- 所有 URL 字段必须通过 HTTPS 访问。

### 3.4 非功能性需求

#### 3.4.1 性能需求
| ID     | 描述                 | 量化指标                            |
| ------ | -------------------- | ----------------------------------- |
| NFR-01 | 图片网格首屏加载时间 | ≤ 2 秒（4G，10 张缩略图）           |
| NFR-02 | 搜索响应时间         | ≤ 3 秒                              |
| NFR-03 | 图片详情弹窗打开时间 | ≤ 500ms（含原图加载，允许渐进显示） |
| NFR-04 | 页面切换动画帧率     | ≥ 30fps                             |
| NFR-05 | 支持并发在线用户数   | ≥ 200（峰值 QPS 10）                |

#### 3.4.2 安全性需求
| ID     | 描述                                                                   |
| ------ | ---------------------------------------------------------------------- |
| NFR-06 | 用户隐私：openid 不得暴露在前端，仅后端使用                            |
| NFR-07 | 通信安全：所有请求强制 HTTPS                                           |
| NFR-08 | 认证令牌：JWT token 有效期 7 天，具备刷新机制（V1 可暂不做）           |
| NFR-09 | 输入校验：前端提交的所有文本字段需过滤 XSS 脚本字符，后端二次校验      |
| NFR-10 | 账号注销：用户注销后，其所有个人数据（收藏、用户信息）从数据库物理删除 |

#### 3.4.3 可用性需求
| ID     | 描述                                                                       |
| ------ | -------------------------------------------------------------------------- |
| NFR-11 | 易用性：首次进入小程序时，首页应有简易引导蒙层（可跳过）                   |
| NFR-12 | 防错：收藏按钮操作需有震动和视觉反馈，避免重复点击；删除类操作必须二次确认 |
| NFR-13 | 可访问性：所有非装饰性图片必须提供 `alt` 文本属性（AI 描述充当）           |
| NFR-14 | 容错：网络异常或服务器错误时，提供重试入口，不出现白屏                     |

#### 3.4.4 可维护性需求
| ID     | 描述                                                       |
| ------ | ---------------------------------------------------------- |
| NFR-15 | 前端组件化：图片网格、图片弹窗、主题卡片抽取为独立组件     |
| NFR-16 | API 版本管理：后端 API URL 包含版本号 `/v1/`               |
| NFR-17 | 日志：后端记录关键操作日志（登录、收藏、搜索），保留 30 天 |

#### 3.4.5 可移植性需求
| ID     | 描述                                                  |
| ------ | ----------------------------------------------------- |
| NFR-18 | 兼容 iOS 12.0+、Android 8.0+、微信客户端 7.0+         |
| NFR-19 | 适配刘海屏、底部指示条，使用 `safe-area-inset-*` 适配 |

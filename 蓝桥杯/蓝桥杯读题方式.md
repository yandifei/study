### 第一步：读题“三遍法”（用时 3-5 分钟）

不要从头到尾匀速读，要像侦探一样分层提取。

**第一遍：只看输入输出格式（30秒）**
直接拉到题目最下方的“输入格式”和“输出格式”。
- **目的**：立刻知道这道题要你做什么类型的事。是求最大值？打印图形？判断 Yes/No？
- **举例**：看到输入是一个整数 N，输出也是一个整数，大概率是**数学规律**或**动态规划**。看到输入是地图（字符矩阵），输出是最短步数，大概率是 **BFS 搜索**。

**第二遍：带着边界看描述（2分钟）**
只看题目的**第一段**和**最后一段**，中间的故事背景（比如“小明在星球上挖矿...”）先跳过。
- **目的**：抓住核心逻辑。
- **划出关键词**：
    - “最大值/最小值” → 贪心、DP、二分
    - “最短路径/最少步数” → BFS
    - “所有方案/组合数” → DFS、排列组合
    - “模 10^9+7” → 说明数据极大，必须用数学或 DP，暴力必超时。

**第三遍：死磕样例解释（最重要的一步）**
把样例输入抄在草稿纸上，对照着题目的规则，**手动推导一遍**样例输出是怎么来的。
- **关键**：如果手算的结果和样例输出对不上，说明你理解错了题意。**千万不要在这个阶段开始写代码**，一定要推到一致为止。
- **目的**：这是帮你理解题目最精确的“翻译官”。

### 第二步：看数据范围定算法（黄金 30 秒）

Python 蓝桥杯 B 组，**数据范围直接决定了你能不能用暴力**。读完题立刻看“数据规模与约定”：

| 数据范围（N 的大小）   | Python 能用的方法           | 关键词联想                 |
| :--------------------- | :-------------------------- | :------------------------- |
| **N ≤ 10 ~ 20**        | **直接暴力枚举 / DFS 回溯** | 全排列、求子集、指数级搜索 |
| **N ≤ 100 ~ 500**      | **三重循环 O(N^3)**         | 简单 DP、Floyd最短路、模拟 |
| **N ≤ 10^5**           | **O(N log N) 排序/二分**    | 前缀和、贪心、双指针、堆   |
| **N ≤ 10^6**           | **O(N) 一次遍历**           | 线性 DP、单调栈、快读      |
| **N 极大但答案是模数** | **矩阵快速幂 / 数学公式**   | 找规律、费马小定理         |

**核心心法**：如果 N ≤ 10，放心大胆用 `itertools.permutations` 暴力枚举，不要想复杂算法。

### 第三步：B 组高频题型的“条件反射”识别法

看到特定描述，立刻联想到对应的 Python 模板库/写法。

**1. 看到“迷宫”、“地图”、“最少步数”**
- **反射**：`from collections import deque`
- **动作**：套 BFS 模板（队列里存坐标和步数）。

**2. 看到“最大价值”、“最长子序列”、“有多少种方法”**
- **反射**：`dp = [0] * (n+1)`
- **动作**：定义 `dp[i]` 的含义，找递推关系。

**3. 看到“区间修改”、“求某一段的和”**
- **反射**：`pre_sum = [0] * (n+1)`
- **动作**：先做前缀和数组，查询直接 O(1) 切片相减。

**4. 看到“模 10^9+7”但数据很大**
- **反射**：这题有数学规律或者公式，暴力会超时。
- **动作**：手算前 5 项找规律（往往是斐波那契或等差等比）。

**5. 看到“只要求输出 Yes 或 No”**
- **反射**：可能是个判断性质的贪心或者并查集。

### 第四步：B 组比赛的特殊生存技巧（今晚最后再看一遍）

1. **善用 Python 标准库**（这就是你 B 组的优势，不用手写数据结构）：
    - 全排列：`itertools.permutations`
    - 组合：`itertools.combinations`
    - 计数器：`collections.Counter`
    - 二分插入：`bisect` 模块

2. **填空题 vs 编程题策略不同**：
    - **填空题**：只填答案，不限制时间。**Python 可以本地跑一年**。如果没思路，直接写暴力 `while True` 死循环枚举，跑出结果填上去完事。
    - **编程题**：注意输入读取效率。建议统一用 `sys.stdin.read().split()` 一次性读完全部数据，最快。

3. **实在读不懂题的“保命招”**：
    - 如果题目文字超过一屏，且样例输入是 `0 1 2 0 1...` 这种长串数字。
    - **做法**：先直接模拟过程。读不懂算法意图，就把题目描述的**动作**用代码写出来（比如：如果遇到0就计数，遇到1就清零），往往这就是正解（模拟题）。

### 今晚的实操建议

拿去年的真题，**只做审题练习，不写完整代码**。
流程如下：
1. 看题目 2 分钟。
2. 关上网页，用笔在纸上写出：这道题输入什么？输出什么？N 最大是多少？我打算用 **哪个库** 的 **哪个函数** 解决？
3. 对照答案验证思路。

拿到卷子，深吸一口气，心里默念：“**样例通不过，代码就不写**”。稳住，B 组比的是细心和对 Python 特性的熟悉，你的优势正好在这里。
